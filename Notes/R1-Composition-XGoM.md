# R1 · X-GoM：跨视角定性关系代数（同款对应 + 缺口组合）

> 状态 ⚪构思 ｜ 工作名 **X-GoM** ｜ 最近更新 2026-06-09
> **定位**：[Spatial-Reasoning.md] 中 R1 的"一次性完整构建·正确性路线"。姊妹方案 [R1-Progressive-ACVR.md]（渐进按需·效率路线），两者共享 pose-free 跨视角对应原语与 GoM"画关系→MLLM 答"接口，分界在构建策略。背景调研见 [论文笔记.md]。
> **一句话**：在逐视角 GoM 之上建一层 **pose-free 的"定性关系代数"**——用共享锚点估**定性参照系变换 τ**，把多个视角 GoM 的关系搬到同一参照系；**同一套 τ+关系机器服务两件事**：(应用①) 当外观分不清同款实例时，用"对锚点的定性关系签名"做跨视角对应；(应用②) 把单视角观测不到的缺口关系组合出来。最终把自洽结构画回像素喂 MLLM 作答。
> **设计原则**：training-free、pose-free、不假设时序；**可靠优先于覆盖**（宁可弃权交 MLLM，绝不自造矛盾）；**MLLM 始终作答**（我们只负责喂好结构，不替它答）。

---

## 1. 动机：两个挑战

MLLM 在多视角空间推理上弱，集中在两个被 All-Angles 实测到的短板，X-GoM 直接以这两点为出发点：

- **挑战① 同目标跨视角关联弱（→ counting 失败）**：模型分不清"视角 i 的某物体"与"视角 j 的某物体"是否同一个 → 把同一物体在各视角重复数。All-Angles 主体多为**人、常同款球衣**，**纯外观匹配必歧义**（典型困境 "Person A/B in View1, C/D in View2"）。
- **挑战② co-visibility gap（共现缺口）**：A 只在视角 i、B 只在视角 j，**没有任何单一视角同时看到 A 和 B**（从不共现）→ 它们的关系任何单视角都观测不到，只能跨视角*推*。这是"局部好 / 全局差"在真·多视角下的正确形式——一个**视角现象**，区别于 VSI-Bench 的*距离*现象（VSI-Bench 单相机室内视频、距离驱动，仅作 eval 广度，不作动机）。

GoM 能把**单视角内**的定性关系图抽好（"局部好"），但对这两点——跨视角的同物体关联、跨视角的关系建立——**单图 GoM 无能为力**。

---

## 2. 核心思想：一个机制，两个应用

- **机制 = 跨视角定性关系代数**：用共享锚点估**定性参照系变换 τ**（只换关系标签、不算数值坐标），把多个视角 GoM 的定性关系搬到统一参照系后做符号运算。
- **应用①（解挑战①）= 关系签名对应**：外观能确定的对应直接用外观；**外观歧义的同款实例**，改用"它对一组已匹配锚点的定性关系向量（τ 对齐后）"当指纹来配。
- **应用②（解挑战②）= 关系组合**：对落在 gap 的 (A,B)，沿锚点路径把各段关系搬到同帧、逐轴符号相加，多路径投票定值。
- **两应用共享同一 τ+关系机器，且互相喂**：对应给出锚点与 τ → τ 帮助签名匹配与关系组合 → 签名/组合反过来改进对应（详见 §3.4 反哺闭环）。

## 3. 方法（training-free）

### 3.0 总览 + 框架图

逐视角 GoM 感知 → 跨视角锚定 + τ 估计 → **[应用① 关系签名精化对应] ⇄ [τ 重估]（反哺闭环）** → **[应用② gap 关系组合]** → 渲染喂 MLLM。

```
[view_1..N] ─GoM感知→ 局部关系子图 {F_v} ─┐
     │ region emb                          ├─锚定+τ→ 全局id + 共享锚 + 定性参照系变换 τ
     └─────────────────────────────────────┘        │
   应用① 关系签名对应(反哺闭环):  外观优先 → 歧义残量用"对锚点关系签名"配 → co-visible自洽校验 → 重估τ ↺
                                                              │ (gid, τ, F)
   应用② gap组合:  gap对(A,B) ─找锚点路径→ 搬到统一帧 → 三轴符号组合 → 多路径投票 →(A,B)关系或∅
                                                              │
   原图 + 全局id标记 + query相关关系子图 ─────────────────────►MLLM─►answer
```

### 3.1 Stage A · 逐视角定性感知（GoM 作感知层）

每视角开放词表检测/分割（Grounding-DINO / SAM / SEEM）→ 抽 GoM 式定性关系：方向（左右/上下）、深度（前后，单目深度）、near。**这些关系只在本视角相机参照系 F_v 成立**。同时取 region embedding 备用。继承 GoM 已验证的"局部好"，不重证单视角感知。

### 3.2 跨视角锚定 + 锚点策略

- **锚点定义** = 在 **≥2 视角有可信匹配**的物体（桥）。τ 估计、gap 路径都踩着锚点走，所以锚点是整套演算的"轨道"。
- **(C) 混合对应**：现成跨视角匹配器（V²-SAM / VICP / MMVM-CoLVA）出**候选** → 用**关系/布局一致性重排 + 软置信**消歧（外观 + 互最近邻 + 循环一致 + 关系图一致）。全程**软对应**带置信，低置信喂下游 UNKNOWN/∅。
- **锚点策略（关键，因 All-Angles 主体多为人且常同款）**：优先选 **distinct / 静态 / 定位良好**的场景物作 τ-锚点/桥（外观就能可靠配上、关系稳定、框准）；**identical/可变形主体（人）作被查询、被组合对象，不当锚点**（人聚集且同款，外观与关系消歧都最难）。一句话分工：**稳定独特的死物当"轨道"，人当"货物"**。

### 3.3 ★ τ · 参照系搬运（两应用共享的核心）

> **"参照系搬运"是什么**：每台相机有自己的"左/右/前/后"，同一物理关系从不同相机看标签会变（正面的"左"从侧面可能成"后"）。**搬运 = 用 τ 把 j 视角坐标系下的关系标签翻译成 i 视角坐标系下的标签**（定性换标签、不算数值；如 τ≈180° → left↔right、front↔behind 互换、上下不变）。拼跨视角关系链前，必须先把各边搬到同一参照系。

用**共享锚点**估两视角间*定性*方位变换 τ(i→j)：

- **共享锚点对 (a_p, a_q)**：指**两个不同物体**、且**在 i、j 两视角都出现**（同全局 id）的一对。m 个共享锚 → C(m,2) 对，每对投一票。每对在两视角各读 左右符号 `s^LR=sign(cx_q−cx_p)`、深度符号 `s^D=sign(depth_q−depth_p)`。
- **真值表**把 (LR, depth) 在 i→j 的变化读成粗方位 bin：左右保持&深度保持 → ≈0°（同侧）；左右翻转&深度翻转 → ≈180°（异侧）；左右↔深度互换 → ≈±90°。
- 所有锚点对**投票** → τ = 多数 bin，置信 = 票差；**票差低于阈值 → τ=UNKNOWN**（该边不参与搬运）。
- **鲁棒退化**：深度只用于把分辨率 2-bin→4-bin；深度不可信时退回**仅左右翻转的 2-bin（同侧/异侧）**，τ 退化不崩。
- **硬性前提（≥2 锚点）**：1 点定不了朝向，2 点粗估，≥3 投票更稳；不足 → τ=UNKNOWN → **多跳绕路**（经中间视角）或交 MLLM。本质是**连通条件**：连接 query 两物体所在视角的链条、每段都要 ≥2 共享锚（锚不必是 query 物体，任意 2 个 distinct 共享物即可）。

### 3.4 应用① · 关系签名对应（解挑战①）

**级联：外观优先、关系兜底。** 关系签名严格架在 appearance 之上（要靠 appearance 先检测分割、定锚、bootstrap τ），且更重，故只在需要时启用：

- **置信门路由**：appearance 决定有把握的（distinct 物体、top1-top2 margin 大、互最近邻+循环一致都一致）→ 直接用外观；**只有外观歧义的残量**（同款实例、近似并列候选）才升级到关系签名。
- **签名**：物体对锚集的**定性关系向量**（到每个锚的 左/右、前/后、near）。跨视角配对时把 V_j 的签名经 τ 搬到 V_i 帧，签名吻合 = 同物（软置信）。
- **co-visible 交叉验证**：在 ≥2 视角都出现的物体，其 GoM 边经 τ 后**必须自洽**；不自洽 → 该对应是外观误配，拒掉/降权。一箭双雕：**清洗对应 + 体检 τ**（已知同物的关系搬不一致 ⇒ τ 错）。
- **反哺闭环（K 轴）**：appearance → 粗 τ → 关系签名/交叉验证改进对应 → 用更净的对应重估 τ → 重复。**K = 迭代轮数**，是消融旋钮而非架构选择：
  - **K=0**：纯前向、无反哺（消融基线，证明"反哺才涨点"）；
  - **K=1**：单次反哺；
  - **K=收敛**：迭代到 (gid, τ) 稳定。
  - bootstrap 破"对应↔τ"鸡生蛋：外观可靠的 distinct 锚先行起步（不需 τ），粗 τ 再解锁同款实例消歧。定性离散信号饱和快，预期 1–2 轮收敛。
- **防漂移（堵评审）**：三道刹车——co-visible 校验是**独立**检验（已知同物必须搬运自洽）＋软置信不硬提交＋**只接受"提升全局一致性"的精化（单调约束）**。
- **(parked) 序结构 / 群组匹配**：当签名撞格时（一群同款实例对所有外部锚落在同一粗关系格）的升级——它们**彼此之间**仍有稳定的左右序，经 τ 可预测变换。本质是**同一个签名操作、把参照集从外部锚扩到群内同伴**（不是第二个机制，故 park/un-park 零架构代价）。**仅当 §7 测出"签名可分率"低（撞格多）才启用**；否则残余撞格 → 软 id/UNKNOWN → 交 MLLM 看像素（诚实弃权，非静默错误）。

### 3.5 应用② · 关系组合（解挑战②）

**分流：co-visible 直答 / gap 才组合。** query 两物体若在某视角共现 → GoM 感知层**直接读**、不触发组合；只有落在 gap 的对才走搬运+组合。好处：增益来源干净（组合只在 baseline 失败处发力），评测可按 co-visible vs gap 分层。

- **三轴符号代数**：每条（已搬运到统一参照系的）关系编码成三轴符号向量：x: left(−)/right(+)；y: below(−)/above(+)；z: behind(−)/front(+)；`near` 单独标记且**跨参照系不变**。每条边通常只点亮 1 个主导轴；两轴超 margin 则点亮 2 轴（如 front-left = x−, z+）。
- **组合（沿路径 A→P→B）= 逐轴符号相加**：`(+)+(+)=+`，`(−)+(−)=−`，`(±)+(0)=±`，**`(+)+(−)=∅`（该轴未定/弃权）**。读出 A→B 符号向量，按 query 关心轴作答。
- **多路径自一致**：A、B 间常有多条锚点路径 → 多个符号向量 → **逐轴投票**。"一致"算法：每轴单独看各路径在该轴给的符号（忽略 ∅）——非 ∅ 票**同号**（或 ≥ 阈值比例，如 ≥⅔）= 一致，取该号、置信 = 票差；**+ 与 − 并存且无明显多数 = 该轴判 ∅**。例 x 轴：`+,+,+`→一致(+)；`+,+,−`→2/3 看阈值；`+,−`→打平→∅。τ 偶发判错被此层抓出。
- "符号相消即弃权"保证**我们喂入的结构自洽（sound）、不自造矛盾**。注意：MLLM 始终作答 → 不对最终答案的 IC 给硬上界，只主张"喂入自洽结构 → 经验性降低 MLLM 的 IC"（机理验证）。

### 3.6 Stage E · 渲染 / 喂 MLLM

沿用 GoM"把关系画上图"的有效接口（实测视觉 SG > 文本 SG +10%），并解决跨视角画不了单根箭头的问题：

- **视角内关系**（A、B 同视角）：直接在该图画 head→tail 箭头 + 标签（与 GoM 一致）。
- **跨视角组合关系（gap，本方法新增）**，两种画法做消融：
  - **(a) 拼图画布 montage（主选）**：N 视角拼成一张大图，**跨格画箭头**（V1 的 #3 → V3 的 #7）。代价：每视角分辨率下降。
  - **(b) 分图 + 锚定文字标签（对照）**：保持多图原样，在物体标记旁写 "← left-of #3(V1)"，附一张极小组合关系图。
- 两种都叠**全局一致 id 标记**（ViKey 规范：数字、底角 BL/BR、勿遮目标，同物体跨视角同 id/同色）；文本三元组仅作补充。
- **MLLM 始终作答**：被卸掉的是"跨视角缝合"（谁是同一物、视角怎么排、全局关系），不是作答本身。**置信/∅ 只决定"断言什么进 prompt"**：高置信关系画出/写出；∅/低置信的边省略或标注不确定，让 MLLM 对这部分回退到原始像素自判。

### 3.7 （可选能力，parked）跨视角共识修复单视角 GoM 错误

view i 的 GoM 说"A 在 B 左"，但 j、k、l 经 τ 对齐后的共识是"A 在 B 右" → view i 这条边大概率是感知错（遮挡/视点/深度）→ **标出并纠正**。这是**单视角 GoM 结构上做不到的**（它没有第二意见），是"多个 GoM"真正非冗余的用法，也是干净的消融（有/无跨视角边修复）。**仅当 §7 测出"单视角 GoM 边错误率"可观才上**；当支撑能力，不作主线。

## 4. 伪代码

```python
def XGoM(views, query, K):
    # ---- Stage A: 逐视角 GoM 感知 ----
    F, emb = {}, {}
    for v in views:
        objs   = open_vocab_detect_segment(v)
        F[v]   = gom_local_relations(objs)         # 方向/深度/near, 仅在 F_v 成立
        emb[v] = region_embeddings(objs)

    # ---- 跨视角锚定 + 应用①: 关系签名对应 (反哺闭环, K 轮) ----
    gid  = appearance_match(emb)                   # 外观优先: distinct 稳, 同款人歧义/未决
    anch = high_conf_distinct_static(gid)          # 引锚: 外观可靠 + 静态 distinct
    tau  = estimate_tau(anch)                      # 粗 τ (≥2 共享锚; 不足→UNKNOWN→多跳)
    for k in range(K):                             # K=0 纯前向(消融) / 1 单次 / 收敛闭环
        for (i, j) in view_pairs(views):
            for o in ambiguous_objs(i, j, gid):            # 仅外观歧义残量(同款实例)
                sig_i = signature_to(o, anch, i, F)        # o 对锚的定性关系向量
                sig_j = transport(signature_to(o, anch, j, F), tau[i, j])
                gid   = match_by_signature(sig_i, sig_j, gid)         # 吻合=同物, 软置信
            gid = reject_inconsistent(covisible(i, j, gid), F, tau[i, j])  # co-visible 自洽校验
        tau = estimate_tau(refined_anchors(gid))   # 用更净对应重估 τ (单调: 只接受↑全局一致)
        if converged(gid, tau): break

    # ---- 应用②: gap 关系组合 ----
    targets = parse_query_objects(query)           # 目标物体 + 目标轴
    rel = {}
    for (A, B) in target_pairs(targets):
        if co_visible(A, B, gid):                  # 分流: 单视角直接读
            rel[(A, B)] = F[view_of(A, B)][A, B]
        else:                                      # gap: 多路径组合
            vecs = [transport_and_compose(p, F, tau) for p in anchor_paths(A, B, gid, max_len=L)]
            rel[(A, B)] = per_axis_vote([w for w in vecs if w])       # 一致→值, 打架→∅

    # ---- 渲染 + MLLM 作答 ----
    rel = {k: w for k, w in rel.items() if confident(w)}   # 只断言高置信; ∅/低置信不画不写
    img = render_marks_and_relations(views, gid, rel)      # 一致 id + 关系画在像素上(见 3.6)
    return MLLM(prompt(query, img, serialize(rel)))        # MLLM 始终作答
```

## 5. 微例（相对方向）

Q：sofa 在 TV 左边吗？sofa 只在 V1、TV 只在 V3、lamp 是锚点。
V1(F1)：`sofa left lamp`；V3(F3)：`lamp left TV`。共享锚 {lamp, plant} 两视角均 `lamp left plant`（不翻转）⇒ τ(1→3)≈同侧 ⇒ 左右搬运取恒等 ⇒ V3 边搬到 F1 不变 ⇒ 组合 `sofa left lamp ∘ lamp left TV` ⇒ **sofa left TV，Yes**。
反例：V3 里 `lamp right plant`（翻转）⇒ τ≈180° ⇒ 搬运交换 V3 左右 ⇒ 组合变 ∅（歧义）⇒ 走另一锚点路径或**诚实报不确定**（比自信错更降 IC）。

## 6. 可靠性与防漂移

### 6.1 τ 可靠性研究（把"启发式脆"变成被证伪的命题）
- **③ 真实数据免费 τ 准确率（先做）**：All-Angles 自带相机位姿/视点排序子任务 → 我们的 τ 标签直接给粗视点序 → 该子任务 Acc **就是** in-the-wild τ 准确率，零额外标注，顺带补位姿短板。
- **② Oracle-τ 差距（最致命一击）**：**只改 τ 来源、其余完全相同**，比三档端到端——no-transport（盲拼，IC 应爆）/ heuristic-τ（我们估的）/ oracle-τ（用 GT 位姿得到的完美 τ，仅合成数据可做，作天花板）。逻辑：heuristic 的 Acc/IC ≈ oracle ⇒ τ 估计误差几乎不影响终效 ⇒ **τ 不脆**；差距大则 τ 是瓶颈。
- **① 受控合成集（后补）**：渲染已知位姿，画 τ-acc vs（锚点数 / 对应噪声 / 基线角）敏感性曲线；预期 ≥3 锚点即饱和、对噪声优雅退化。
- **④ 自一致安全网**：个别 τ 判错被多路径一致性抓出 ⇒ **端到端鲁棒性 > 裸 τ 准确率**（"τ 不需完美，一致性层吸收其错误"）。

### 6.2 弃权与 MLLM 兜底
- **可靠优先于覆盖**：高置信关系才断言进 prompt；∅/低置信的边省略或标注不确定，MLLM 回退到原始像素自判。
- **对照协议 = All-Angles 原始强制作答**（MLLM 每题都答），与 baseline 同协议可比。
- 取代旧"弃权曲线"的消融：`置信阈值（断言多少边）vs Acc` 曲线——权衡"多给结构" vs "少给噪声"。
- **关于 IC**：不测 MLLM 原生 IC（已知很高），而看"喂入自洽结构后 IC 是否下降"，作机理验证（不主张对答案 IC 的硬上界）。

## 7. 诊断电池（先测后定——用数据决定加不加机制，而非直觉）

在 All-Angles 上用 GT（EgoHumans/Ego-Exo4D 本身带多视角人物 id）跑三个廉价的数，直接拍板几个开放设计：

| 诊断量 | 怎么测 | 决定什么 |
|---|---|---|
| ① **签名可分率** | 需消歧的同款实例对里，多少被"对锚点的定性关系签名"分开 / 多少撞格 | 签名够不够；**要不要 §3.4 的序结构**（撞格多才 un-park） |
| ② **同款歧义占比** | counting 中外观无法区分的实例占比 | 应用①（关系签名分支）的作用域 / headline 增益上界 |
| ③ **单视角 GoM 边错误率** | 单视角 GoM 关系与跨视角共识的矛盾比例 | **要不要 §3.7 的边修复**（占比可观才上） |

## 8. 实验设计

- **基准**：All-Angles（主，盯 IC + counting）+ VSI-Bench（次，当无序多视角集、丢帧序、只承担 Acc 广度）。注意 VSI-Bench 上 Coarse Corr 是头号 baseline（其主场）。Ego3D 暂 park（几何性弱适配，留给 C/U 系列）。
- **Baselines**：原始多图直问 / SoM / **Coarse Corr（纯 id）** / **GoM 各视角并集（= 朴素拼接，关键消融：证明 τ+组合/签名才涨点）** / Ego3D-VLM / All-Angles Identification-CoT / MvMHAT 类（对应侧对照）。
- **指标**：Acc（主）+ IC（机理验证）+ 子任务（counting / camera-pose / relative-direction）；**按 co-visible vs gap 分层**。
- **关键消融**：
  - 应用②：去搬运（直接组合）→ IC 应爆；去组合（只并集）；单路径 vs 多路径；路径长 L。
  - 应用①：纯外观对应 vs 外观+关系签名兜底（隔离关系贡献）；**反哺 K 轴：K=0 / 1 / 收敛**；锚点数敏感。
  - 渲染：标记位置（ViKey 底角 vs 顶角）；montage(a) vs 文字标签(b)；数字 vs 文本 id。
  - τ：见 §6.1 的 oracle-τ / 免费 τ / 合成敏感性。

## 9. 计算成本（推理开销）

- 一次性辅助前向：每视角 1×检测/分割 + 1×特征（DINOv2-L）；对应 O(N²·#obj)；τ 估计 O(N²·#anchor²) 极廉价；深度仅需相对深度（MiDaS 级）。
- LLM 侧：标记叠在图上**不增视觉 token**；组合关系是小段文本（几十~数百 token）；reason-only，**视觉 token 数不变**。净增 ≈ 辅助一次性前向 + 适度输出增长。

## 10. 相关方法与切割（事实记录 · novelty 措辞另定）

> 仅记录最接近的近邻与**事实差异**，供日后写作选 claim。诚实红线：**不宜声称"首个用空间关系做跨视角关联"**（学习/度量版先例已存在）；**邻居消歧同款外观的动机非全新**（group-context re-id、体育 constellation 早有）。

| 工作 | 它是什么 | 与 X-GoM 的事实差异 |
|---|---|---|
| **Coarse Corr** (CVPR'25, 2408.00754) | training-free + 视觉提示 + 跨视角对应，pose-free | 只画**实例 ID**（靠 tracking），无关系场景图、无 τ、无组合；本质时序 tracking，未测静态多相机 |
| **Graph-of-Mark** (AAAI'26, 2603.06663) | 单视角场景图视觉提示，training-free（本方法母体） | 严格**单视角**，无 multi-view/cross-view、无 τ、无跨图配对（无任何多视角 follow-up） |
| **Elich et al.** (ICRA'23, 2305.02398) | 学习型 GNN，用物体间空间关系 + 外观跨视角匹配 | **学习型**、关系是**数值几何**、含外观、异质物体（不针对同款歧义）、无显式 τ |
| **Gerke et al.** (CVIU'17, S1077314217300723) | 用球员对队友的相对位置 constellation 消解同款球衣 | **单视角→身份库识别**（非跨视角对应）、**度量直方图**（非定性）、**需训练**、隐含 pitch registration |
| **APC / Perspective-Aware** (2504.17207) | training-free 场景抽象 + 视角变换（τ 概念最近） | 依赖**朝向/深度几何估计**、不用视觉提示、偏单场景视角想象 |
| **MvMHAT** (TPAMI'24, 2401.17617) | 无标定跨视角多人关联（任务最像） | 靠**学外观嵌入 + 时空自一致**、**需训练**；不显式处理同款关系消歧 |
| **MSG / Multiview Scene Graph** (2410.11187) | pose-free 跨视角物体关联，建拓扑图 | 靠 **DINOv2 外观 + 对比学习**、**明确不处理**同款外观歧义、边是拓扑非定性方向 |
| **Sparse3DPR** (2511.07813) | training-free 稀疏多视角场景图 + 子图推理 | 场景图作**文本**喂纯 LLM（非视觉提示）、走重建/层次解析 |
| **ViewFusion** (2603.06024) | 多视角推理：先推视角关系与变换再答 | **需训练**（合成监督 + GRPO）、无视觉提示、无显式场景图 |

补充（QSR 工具）：定性方位演算（cardinal direction calculus、RCC 等）是成熟符号框架，但应用集中在 GIS/机器人导航/时空数据库，**未见用于多相机/体育的跨视角实例对应** —— τ 是把成熟 QSR 嫁接到这座桥的无参数算子，三路查新均未见同类。

## 11. 目标基准画像（All-Angles，2504.15280）

- 90 真实场景、>2100 人工 QA；源自 **Ego-Exo4D + EgoHumans**（真实同步多机位人类活动：篮球/足球/烹饪/演奏；办公室/健身房/厨房/操场）。
- 每场景 ≥3 视角、精选 4–5 视角；**多数任务用随机 2 视角**，**counting / camera-pose 用全部视角**（→ 多路径自一致主要在全视角任务生效；2 视角任务若有 ≥2 共享锚仍可多路径）。
- 6 任务：counting / attribute / relative distance / relative direction / object manipulation / camera pose。
- 作者实测首要失败 = **部分遮挡下的跨视角对应 + 粗相机位姿**（X-GoM 直击点）。
- **重复/同款实例普遍且核心**：counting 即"跨视角不重不漏地数"，主体多为人、常同款球衣 → 纯外观必歧义 → 坐实必须上关系消歧（应用①）。
- **任务-组件映射**：counting ↔ 一致全局 id 去重 + 关系签名对应（应用①，最吃对应误差）；relative direction ↔ 方向组合（应用②核心）；relative distance ↔ near/组合；camera pose ↔ τ 标签（免费 τ 验证）；attribute ↔ 逐视角感知 + 一致 id。
- **隐忧**：开阔运动场景（球场）**distinct 静态锚点稀少** → τ 可能缺锚 → 退 UNKNOWN/降级；列为 limitation，应实测占比。
