## 1. Thinking in Space: How MLLMs See, Remember, and Recall Spaces

- **arXiv**: 2412.14171 (v2)
- **作者**: Jihan Yang, Shusheng Yang, Anjali W. Gupta, Rilyn Han, Li Fei-Fei, Saining Xie
- **主题**: 提出 VSI-Bench 评测多模态大模型（MLLM）的视觉空间智能，并剖析模型"在空间中如何思考"
- **本次记录范围**: 第 5 章（语言层面）+ 第 6 章（视觉/记忆层面）
- **探针实验对象**: VSI-Bench 上表现最好的模型 **Gemini-1.5 Pro**

### 第 5 章 How MLLMs Think in Space Linguistically（语言层面）

核心做法：让 Gemini-1.5 Pro 用语言讲出自己的内部推理（self-explanation），再人工分析。

**5.1 自我解释探针（Probing via Self-Explanations）**
- 方法：抽 163 个错误答案，让模型解释，人工逐条审阅。
- 模型已具备的能力：视频理解强（时间戳描述准）、会做合理分步推理、甚至会**自发构建全局坐标系 → 暗示存在隐式 world model**。
- 典型失败：**自我中心↔异我中心视角转换错误（egocentric-allocentric）**、**关系推理错误**（如顺着第一人称视角答"右转"，未建立客观布局）。
- 错误分四类：① 视觉感知 ② 语言智能 ③ 关系推理 ④ 视角转换。

> ⭐ **结论 1**：约 **71% 的错误源于空间推理（③+④）→ 空间推理是 MLLM 的首要瓶颈**。模型的"看"和"语言"基本没问题，卡在把所见组织成正确空间关系 + 视角转换。

**5.2 CoT 类方法的局限（Limits of CoT Methods）**
- 测试 3 种主流提示技术：Zero-Shot CoT、Self-Consistency w/ CoT、Tree-of-Thoughts (ToT)。
- 结果：三者在 VSI-Bench 上**平均都降低性能**。
  - Zero-Shot CoT 与 ToT 平均 **-4%**；Self-Consistency **-1.1%**。
  - 房间大小 / 物体大小任务最惨，**-8% ~ -21%**。
  - 仅"出现顺序""绝对距离估计"略升（因含较多语言类错误）。
  - 对照：同样 CoT 在通用视频基准 **VideoMME 上反而 +1.6%**（500 题子集 77.2 → 79.8）。

> ⭐ **结论 2**：**语言类提示技术对空间推理有害**；**VSI-Bench 无法靠单纯增强语言能力解决——空间智能与语言能力正交、独立**。

### 第 6 章 How MLLMs Think in Space Visually（视觉/记忆层面）

用**认知地图（cognitive map）**作探针，研究模型如何"记住"空间。

**6.1 认知地图探针（Probing via Cognitive Maps）**
- 方法：让模型在 **10×10 网格**中预测各物体中心位置，画出脑内空间表征。
- 评估：两两物体欧氏距离与真值对比，误差 ≤1 格算对。
- 发现：相邻物体定位准确率 **64%（局部感知强）**；距离越远准确率急剧下降（**全局感知差**）。

> ⭐ **结论 3**：**MLLM 记忆空间时构建的是"一系列局部世界模型"，而非统一的全局模型**。从离散视频帧拼出全局空间表征本身就难。

**6.2 用认知地图改善距离推理（Better Distance Reasoning via Cognitive Maps）**
- 做法：让模型先生成认知地图、再用它回答"相对距离"任务。

| 设置 | 相对距离准确率 |
|---|---|
| 不用认知地图（基线） | 46.0 |
| 用模型自生成地图 | **56.0（+10）** |
| 用真值 GT 地图 | **66.0（+20）** |

- 画布尺寸：MLLM 10×10=56.0、20×20=54.0；GT 10×10=66.0、**GT 20×20=78.0**。

> ⭐ **结论 4**：先建认知地图再推理 **+10pts**；用准确 GT 地图增益 **+20~32%**，说明"建出全局拓扑一致的准确心理地图"至关重要，但只是拼图一块。**把"构建空间心理世界模型/认知地图"作为 pretext task 是有前景的方向**。

### 总体关键结论
1. **瓶颈在空间推理本身**（关系推理 + 视角转换 ≈71% 错误），不在"看"或"语言"。
2. **纯语言路线走不通**：CoT/ToT/Self-Consistency 对空间任务有害 → 空间智能与语言能力正交。
3. **记忆是"局部拼图"**：能记住局部（相邻 64%），建不出统一全局模型。
4. **视觉化路线有希望**：显式认知地图能切实提升空间推理（+10pts，GT 可达 +20~32pts）。
5. **方法论**：模型有隐式 world model 迹象（自发建坐标系），但心理意象尚不可靠，需专门机制强化。

### 精读 1：Geometry-Guided 3D Visual Token Pruning for Video-Language Models（Geo3DPruner）
- arXiv:2604.18260（2026-04）｜北航/中关村学院｜基座 Video-3D LLM（LLaVA-Video 7B + Qwen2-7B + SigLIP）｜几何编码器 VGGT-1B（冻结）｜代码称将开源。状态：预印本。

**Motivation**：3D 场景被表示成"空间视频"（图像序列+depth+pose）喂给 VideoLM，但视觉 token 量巨大（16 帧=3136、32 帧=6272）成为推理瓶颈。现有剪枝（文本引导 FastV / 视觉引导 VisPruner）只在单帧或时序相邻帧操作，存在两大盲点：① 忽视**多视角一致性**（同一物体在任意帧反复出现的全局冗余去不掉）；② 忽视**保留 token 的空间多样性**（token 扎堆同一物体，object-centric 任务一降预算就崩）。

**创新性**：① 首次用 3D 几何编码器的**跨帧全局注意力图**度量任意帧间几何相关性来去多视角冗余；② **两阶段剪枝**——Intra-Voxel 视角一致性剪枝(VCP) + Inter-Voxel 空间多样性剪枝(SDP)；③ **training-free** 即插即用推理模块。

**方法**：F_s = SigLIP 语义特征 E_s + VGGT 几何特征 G_s；用 VGGT 预测的相机参数+深度做**逆投影**把每个 token 落到 3D voxel（δ=0.1m）。① **VCP**：voxel 内按"收到的平均 incoming attention"打分，每 voxel 留 top-α=50%（去同一 3D 位置的多视角冗余）。② **SDP**：把 voxel 选择形式化为预算内子集选择；用**迭代启发式**（每轮选 top-K=8 最显著 voxel→只在未选 voxel 间重算注意力→反复）抑制同一物体的注意力垄断，逼出空间多样、覆盖全场景的 voxel 子集。剪枝在送入 LLM 前的视觉 token 层完成。

**有趣实验结果**（ScanNet 系 5 benchmark/9 指标，Avg=保留性能%）：
- 16 帧：剪 60%/80%/90% → Geo3DPruner 98.9%/96.1%/**92.1%**，全面领先 FastV(81.0% @90%)、VisPruner(84.9% @90%) 约 7–10pts。
- 32 帧剪 90%：92.0% vs 82.4%/84.8%。
- **固定 token 预算下反超基座**：8 帧不剪=100% 基准时，16 帧剪 50%→**108%**、32 帧剪 75%→**109%**（多采帧+几何剪枝 > 少采帧不剪）。剪 20% 时已略超未剪枝基座。
- 消融：VCP+SDP=94% vs 仅 VCP 89.4%/仅 SDP 91.3%；迭代 SDP 94% vs 随机/均匀 85%；注意力法 94% vs 相似度法 89.9%。
- ⚠️ **全文未报告 FLOPs/延迟/显存/加速比**，"效率"仅以性能-token 曲线体现，且额外引入 VGGT-1B 开销未量化。

**优点**：问题定位新颖精准；极端剪枝(90%)仍保 92%、一致领先；training-free 即插即用；消融充分（尤其"反超基座"发现）；任务覆盖广。
**不足**：① 缺真实效率量化（最大短板，还多引入 VGGT 开销）；② 重度依赖几何编码器质量，仅 ScanNet 室内验证，室外/动态/弱纹理未测；③ 关键超参(δ/α/K)固定无敏感性分析，SDP 为启发式无最优保证；④ 仅单一基座；⑤ 两阶段强耦合、配比敏感，8 帧弱上下文时收益受限。

### 精读 2：Cog3DMap: Multi-View Vision-Language Reasoning with 3D Cognitive Maps
- arXiv:2603.23023（2026-03）｜POSTECH/KAIST/RLWRLD｜骨干 Qwen3-VL-8B(/4B)｜几何模型 Point3R（冻结）。状态：预印本。

**Motivation**：多视角图像的精确空间理解对 MLLM 仍难——视觉表征以语义为主、缺显式几何 grounding。现有"给 token 加几何线索"的方法（VG-LLM、Spatial-MLLM、VLM-3R…）仍要 MLLM **隐式推断** 3D，且重叠视角产生大量代表同一位置的冗余 token，模型需先解纠缠才能推理（绝对距离任务尤差）。

**创新性**：① 提出**递归构建的显式 3D memory = 3D Cognitive Map**，每 token grounding 到唯一 3D 坐标且带语义+几何特征，直接交给 decoder 在结构化地图上推理（而非隐式推断）；② 语义(复用 MLLM 自带 ViT，天然语言对齐)+几何(Point3R)融合 `v_k=f_k+Prj(g_k)`；③ 每个 3D 坐标唯一 token 的**非冗余紧凑表示**（兼具精度与压缩）；④ VSTI/VSI-Bench SOTA。
- **与 Thinking in Space 的关系**：把 VSI-Bench 提出的"认知地图"从行为学探针概念，做成**可计算的显式递归 3D token memory**（computational counterpart），并用它刷 VSI-Bench。

**方法**：递归逐图更新 M_{n-1}→M_n：① Point3R 条件于旧几何特征预测当前 pointmap/几何特征；② 冻结 ViT 抽语义；③ **Memory Update**：按 3D 距离阈值 δ 把旧 token 分为"更新(重叠,用邻域新 token 均值刷新)/保留(远离,原样)/新增(全新区域)"，保证每坐标唯一 token。最终融合 v_k 送 decoder；3D 位置用 `<|pointer_pad|>`+1D-RoPE（**不用 3D-RoPE**，靠几何特征隐式编码）。**需训练（两阶段，非端到端）**：冻结 Point3R+ViT，只训 decoder+projector；VSI-Bench 约 40 GPU·h。

**有趣实验结果**：
- VSTI-Bench：Cog3DMap-8B 比前 SOTA VLM-3R-7B **+8.7%p**，相机运动预测 **+27.5%p**。
- VSI-Bench：平均 **65.1**，比 VST-7B(61.2) **+3.9%p** 创 SOTA；Route Plan 85.6（VST-7B 仅 55.6）。
- RoboFAC（token 效率）：长视频 token **少 90.2%** 而精度持平；中段少 68.9% 且 +2.1%p。
- 消融：几何融合使 CIDEr 65.2→**79.4(+14.2)**；**在几何特征上再加任何 3D 位置编码反而降点**（说明几何特征已隐式含 3D）；3D-REPA(隐式)32.2 < VGM-Aug(拼接不压)59.5 < Cog3DMap 79.4。

**优点**：概念落地清晰、直击"隐式推断"痛点；双 benchmark SOTA；非冗余设计带显著 token 压缩(长视频-90%)且不掉精度；训练代价低、复用自带 ViT、可迁移；消融+注意力可视化可解释。
**不足**：① 高动态场景失效（多物体经过同一位置会互相覆盖特征，作者自承）；② 非端到端两阶段训练；③ 依赖 Point3R 几何质量，误差传导未量化；④ 部分分项未全面领先(Room Size 低于 VST-7B)；⑤ 依赖图像序列顺序，超长序列递归误差累积未分析。
**是否涉及压缩/加速**：**两者兼有，但以提升空间推理精度为首要目标**；token 压缩是非冗余设计的次要收益（只报 token 数，未报 FLOPs/时延）。

### 精读 3：LangDC（Seeing More, Saying More: Lightweight Language Experts are Dynamic Video Token Compressors）
- arXiv:2509.00969｜**EMNLP 2025**｜南科大/港大等｜基座改造自 VideoGPT+｜代码已开源。状态：已发表。

**Motivation**：LVLM 处理大量视觉 token 计算成本高（二次复杂度）。现有压缩器(Q-Former/Resampler/pooling…)用**固定压缩率**，忽略片段间语义密度差异 → 信息丰富片段 token 不够 + 静态片段浪费算力。Oracle 实证：理想 token 数样本间差异极大(156→758)。

**创新性**：① **LangDC**——用轻量语言模型"描述"片段，把预测文本 token 的隐藏状态当 soft caption token 作为压缩表示，**由句末 token 何时出现动态决定压缩率**（"看得多就说得多"）；② **语义密度感知监督**（强 LVLM 教师生成长度自适应 caption 作重建目标）；③ 比 VideoGPT+ **FLOPs -49%** 且性能竞争力。

**方法**：Token pruner = CapPruner(轻量语言专家, **Qwen2.5-0.5B** + language-modeling head 控长度 + post-projector 对齐 LLM embedding；取**第 15 中间层**隐藏状态) + BasePruner(自适应 mean pool)。CapPruner 自回归描述片段，soft token 数随场景丰富度变化（生成式，非聚合/选择）。监督：LLaVA-OneVision 教师抽关键视觉线索 → Qwen2.5-7B 精炼成长度匹配信息密度的 caption。**三阶段训练**（跨模态预训练→CapPruner 预训练/蒸馏→LoRA SFT），非纯端到端。16 帧切 4 段，每段≤128 token。

**有趣实验结果**：
- 效率：FLOPs 49.85T→**25.15T(-49%)**，MVBench 仅 -1.6%；视觉 token 3328→约 **1068**（变长）；显存/延迟均低于 pooling。
- 主结果(3B vs VideoGPT+ 3.8B)：Video-MME 带字幕 51.3(+1.4%)，开放式 MSVD-QA 74.0(+1.6%)、TGIF-QA 76.8(+2.2%)。
- vs 压缩器(MVBench)：LangDC 约 1068 token 达 54.52，接近 3328 token 的 2×2 池化(55.37)，远超同量级 8×8 池化(208 token,49.50)。
- 动态性证据：各任务平均 token 143–257 不等（推理密集任务分更多 token）。
- 消融：去 CapPruner 预训练 54.52→45.40（暴跌）；caption 监督仅 +0.14（贡献有限）。

**优点**：真动态压缩率（机制优雅+定量证据）；效率-性能权衡好(-49% FLOPs)；soft caption token 信息密度高于同量池化；即插即用兼容不同 base pruner；监督信号设计巧妙省标注。
**不足**：① 仅 1.5B/3B 规模，缩放未知；② single-ratio 实现限制部分任务适配(自承)；③ 引入额外小 LLM+三阶段训练+教师蒸馏，pipeline 重；④ caption 监督增益微弱(+0.14)，主收益或来自架构；⑤ 帧数/段数固定(16 帧/4 段)，长视频粒度受限。
**单流 vs 多视角**：**处理单流时序视频**，与多视角无关。思路可迁移（把每视角当"片段"按信息密度分配 token），但需解决多视角空间几何/重叠冗余、视角一致性、多视角联合监督——论文均未涉及。

### 场景已明确、是否精读待定

- **EgoExoMem**（arXiv:2605.18734，2026-05，预印本）——**研究场景**：对**同步采集的第一人称(egocentric)+第三人称(exocentric)视频**做**跨视角记忆推理**。本质是一个 **benchmark**（2.6K MCQ，含时序/空间/跨视角 8 类 QA），附带一个方法贡献 **E²-Select**——面向同步 ego-exo 视频的 training-free 帧选择（基于相关性的预算分配，属帧级压缩）。→ 是"多视角(ego+exo)"且含帧选择，但**主体是评测而非压缩方法**。
- **EventPrune**（arXiv:2605.19506，2026-05，预印本）——**研究场景**：**第一人称(egocentric)动态空间推理** + **事件相机(event camera)** 模态。针对 Video-LLM 密集视觉 token 的二次注意力开销，提出 **Event Cascade Pruning (ECP)**——首个用事件相机高频运动线索作连续运动先验来指导 token 剪枝的 **training-free** 框架。→ 是**推理加速/token 剪枝方法**，但为**单流第一人称+事件相机**，**非多视角**；新意在引入事件相机模态。

### 精读 4：Point3R（Streaming 3D Reconstruction with Explicit Spatial Pointer Memory）
- arXiv:2507.02863｜清华自动化系｜**3D 重建模型（非 MLLM）**。**血缘**：是 Cog3DMap 冻结使用的几何/记忆骨干；其"显式空间指针记忆 + δ 融合"是「多视角 token 压缩」思想的**真正发源地**。

**Motivation**：流式 3D 重建中，Spann3R 隐式 KV 记忆**冗余**、CUT3R **固定长度** token 记忆容量饱和会**丢早期帧**。痛点=记忆隐式、与 3D 结构脱钩。受人类"空间化记忆"启发，让记忆直接锚定在场景 3D 结构上，使存储量随**已探索空间**而非帧数增长。

**创新性**：① **显式空间指针记忆**——记忆=一组指针，每个 = (全局 3D 位置 P, 空间特征 M)，M=视觉外观+几何 特征相加；② **3D 分层位置编码 HRoPE**——把 RoPE 推广到连续 3D 坐标，用 4 个频率基(10/100/1000/10000)分层；③ **简单有效的几何融合**（按 3D 距离+变化阈值 δ 合并/新增），保证指针空间均匀紧凑。

**方法（重点）**：
- **数据结构**：指针 (P, M)。新指针位置 = 对应 patch 内像素预测的全局 3D 坐标取平均；M = Encoder_f(视觉特征) + Encoder_geo(几何)。第一帧坐标系 = 全局坐标系。
- **流式融合规则**：新帧产生新指针 → 找最近已有指针，**距离 < δ → 合并（位置与特征都取算术均值，Eq.10）**，否则新增。
- **δ 是自适应的（关键，A.1）**：δ 随当前记忆指针的空间包围盒尺度变化（每维约切 20 份），场景越大合并越激进 → 指针空间均匀、密度近似恒定。
- **规模**：记忆**随"已探索空间体积"增长，非随帧数线性增长**（亚线性/近似有界，但**非严格固定上限**）。
- **训练**：8×A800、15 天；DUSt3R 权重初始化；三阶段（第一阶段先关融合稳定训练）。

**有趣结果**：长序列(500–1000 帧)重建**碾压 CUT3R**（7-scenes Acc 0.071 vs 0.238）→ 显式空间记忆比固定 token 更能留住早期信息；无序输入鲁棒；vs MASt3R-SLAM **峰值显存更低(5.46 vs 7.18 GB)但逐帧更慢(0.20 vs 0.11 s)**；消融：去 HRoPE 明显变差，**去融合精度反而略升**（即融合是"以极小精度换效率"的有意折中）。**相机位姿是明显短板**。

**优点**：显式可解释、天然处理动态/无序输入；长序列不丢早期信息；真在线流式无需全局对齐后处理；显存较低；δ/l 提供"记忆规模 vs 精度"可调旋钮。
**不足**：① 位姿弱；② 记忆**非严格有界**（探索越大指针越多，且反过来干扰位姿）；③ **均值融合抹平细节**（反光/遮挡/动态物体污染严重）；④ 强依赖自身几何&位姿预测质量；⑤ 逐帧 0.20s 开销；⑥ 首帧参考依赖。

**与本课题（多视角压缩/推理加速）相关性 & 建议**：
- (a) **指针记忆本质就是"以 3D 位置为索引的多视角观测压缩"**——同一 3D 位置的多帧观测被合并进一个指针，token 数与**场景空间体积**挂钩而非帧数。
- (b) **可借鉴设计**：① 用 **3D 位置而非特征相似度**作去重键；② **自适应 δ = 天然的"压缩率旋钮"**（直接决定喂给 MLLM 的 token 数）；③ 几何+外观双通道特征；④ 现成 3D HRoPE；⑤ 流式增量压缩，契合长视频/多视角流。
- (c) **与 Cog3DMap 的关系**：Cog3DMap 的"按 3D 距离 δ 合并 token"（Eq.8–12）与 Point3R 的 memory fusion **机制同源**（最近邻<δ→取均值合并，否则新增）。**关键差异**：**Point3R 用自适应 δ，Cog3DMap 退化成 `pre-defined`（固定）δ** → Cog3DMap 丢掉了"压缩率随场景尺度自适应"的能力，**这是一个可改进切入口**。
- (d) **风险**：记忆非严格有界（token 预算不可预测）；均值融合在动态/遮挡场景失真；对 Point3R 几何&位姿误差强依赖；逐帧延迟。

**由此提炼的课题方向（来自 Point3R 暴露的弱点）**：
1. **找回自适应 δ**：把 Cog3DMap 退化掉的自适应压缩率找回来，甚至做成 query/任务感知的动态 token 预算（呼应 LangDC 的"按语义密度分配"）。
2. **改进融合算子**：用注意力加权/置信度加权/动态场景按时间分桶，替代简单算术平均（见下方"均值融合失真"分析）。
3. **几何骨干开销**：现有方案（VG-LLM/Cog3DMap/Point3R）都白嫖一个 VGGT/Point3R 前向却不计入效率账 → 用更轻量几何代理 / 蒸馏到 MLLM 自身 ViT / 仅用位置做合并键，是"真加速"的关键（详见下方建议）。

## 课题方法分析（基于精读的技术洞察）

> 2026-06-04。两条贯穿 Point3R / Cog3DMap 这条压缩路线的核心问题与改进方向，供方案设计参考。

### A. 「均值融合的失真」：5 类表现与劣势

**机制**：Point3R 与 Cog3DMap 合并 token 时（最近邻 < δ），对**位置和特征都取算术平均**（两篇均 Eq.10：`p'=(1/K)Σpᵢ`，`m'=(1/K)Σmᵢ`）。"失真"= 这种**无差别等权平均**的信息损失与伪影：

1. **视角相关外观被抹平**：同一 3D 点跨视角看不同（镜面高光/反光/各向异性材质/光照/遮挡上下文），平均成"任何真实视角都不存在的糊状中间值"。→ 玻璃/金属/屏幕等视角敏感物体的判别线索丢失。
2. **动态内容碰撞（两篇都承认的硬伤）**：不同物体在不同时刻经过同一 3D 位置 → 落同一指针被平均成"人+车+背景"混合体，轨迹/身份丢失。Cog3DMap 原文："subsequent aggregation **overwrites** features at shared coordinates"。→ 二者都不敢碰高动态场景的根因。
3. **位置平均模糊几何边界 / 吞细结构**：有效空间分辨率被 δ 截断；物体边缘、细长结构（桌沿/椅腿/电线）被并到一起 → 边界涂抹、细结构消失粘连。
4. **等权重忽略观测质量/置信度**：模糊/远/斜视/被遮挡的坏观测与清晰正面特写**同权**，几个坏视角即可把合并特征带偏；无置信度加权。
5. **丧失多视角互补性（对本课题最关键）**：多视角价值=互补信息，均值把互补坍缩成一个平均 token，**主动丢掉多样性**。与 Geo3DPruner 强调"保留 token 空间多样性"形成讽刺对照。

**实证佐证**：Point3R 消融（Table 7）去掉融合精度反而略升 → 等于承认融合是"用精度换紧凑/效率"的有意折中，失真真实存在。

**改进靶点**：把"算术平均"换成**信息保留型聚合**——注意力/置信度加权融合、保留 K 个代表性 token 而非压成 1、动态场景按时间分桶（同位置不同时刻不合并）、对视角敏感物体降低合并强度。

### B. 「几何骨干开销」轻量化：4 条方向

**关键前提**：**合并只需要"位置"，不需要"重建级几何特征"**——δ-合并本质是按 3D 位置分桶去重，只需够一致的坐标。"压缩用的几何"与"提精度用的几何"可解耦，前者要求低得多。
- **建议第一个验证的消融**：把"几何特征参与融合"换成"仅用位置做合并键 + 语义特征来自 MLLM 自带 ViT（免费）"，看精度掉多少；掉得少则整条重几何骨干可拆。

| 方向 | 做法 | 收益 | 风险 |
|---|---|---|---|
| **A. 蒸馏轻量 3D 头（推荐优先）** | 在 MLLM 已有 ViT 特征上挂轻量 head（DPT-lite）预测 pointmap/depth/cam，VGGT/Point3R 作教师蒸馏 | ViT 前向本就要付，只多跑小 head，省掉 1B 级几何模型 | ViT 特征语义导向、几何精度低（但只需够分桶即可） |
| **B. 稀疏化 + 摊销** | 几何骨干只在稀疏关键帧跑，其余帧用光流/匹配/位姿 warp 传播 | 几何成本 O(帧数)→O(关键帧数)，砍掉 VGGT 全局 attention 大头 | 传播误差累积；动态场景 warp 失效 |
| **C. 几何感知统一编码器（最干净最难）** | 给 ViT 加辅助几何目标（蒸馏自 VGGT）微调，一次前向同出语义+粗几何 | 根除"两个大编码器"，终局形态 | 需重训/微调 ViT，可能伤通用语义；工程量大 |
| **D. 有传感器就别学** | 具身/AR/自驾里位姿+深度由 SLAM/IMU/深度传感器免费给 | 几何骨干开销归零 | 仅限有传感器的部署；纯网络视频不适用 |

**真正贡献点**：现有 4 篇（VG-LLM/Cog3DMap/Point3R/Geo3DPruner）**无一把几何骨干开销计入端到端效率账**。无论选哪条路，先老实测出"几何骨干吃掉多少 FLOPs/延迟/显存，轻量代理把它从 X 降到 Y"，本身就是能立住的效率结果（现为空白）。

**落子顺序**：① 做"仅位置分桶"消融（验证前提）→ ② 走方向 A（蒸馏轻量 3D 头）作主线 → ③ 方向 B 进一步摊销 → ④ 全程用"含几何开销的端到端 FLOPs/延迟"作评测主轴。

## 研究方向提案 v1：静态多视角图像｜空间推理提升 + 推理加速

> 2026-06-04。目标基准为 **Ego3D-Bench**（同步多相机多图）与 **All-Angles**（多机位静态多图），**均非视频**。本提案的总纲：**只利用跨视角空间结构，不假设任何时序**，方法普适到任意多视角图像集合（驾驶环视 / ego-exo / 无序多机位），退化到单图即标准流程，扩到视频时"每帧=一个视角"可套用但不被时序绑死。

### 基准事实卡
- **Ego3D-Bench**｜arXiv 2509.06266｜ICLR 2026｜华为。ego-centric 室外**动态**、6/7/5 路同步相机的 **multi-image（非视频）**；8600+ QA；5 任务：绝对距离 / 相对距离 / 定位 / 运动推理 / 旅行时间。SOTA <60%，人类 85.3%。**瓶颈是 3D 感知+推理而非 2D 感知**（给 GT 认知图 → 79.4%）。配套方法 Ego3D-VLM（training-free 文本认知图：REC + 度量深度 + 反投影）：多选 +12%、绝对距离 RMSE −56%；显存 +0.6%、延迟 +31%（延迟主要来自推理输出变长，非视觉 token 变多）。已知缺陷：REC 返回所有表达式框 → 认知图冗余误导。
- **All-Angles**｜arXiv 2504.15280｜AAAI 2026｜Berkeley/NYU（Yi Ma 组）。同场景多机位**静态多图**，每场景 ≥3 视角；2132 QA；6 任务：计数 / 属性 / 相对距离 / 相对方向 / 物体操作 / 相机位姿。SOTA ~55%，人类 82%。三大失败：① 跨视角对应失败（计数把同一物体在各视角重复数）；② 相机位姿估计失败（人类 88.9% vs 模型落后 50+ 点）；③ 跨视角不一致（配对题 **IC**，GPT-4o 相对距离 IC≈70%）。作者结论：纯 prompt 不够，需空间感知模块/训练。**核心指标 IC（配对题不一致率）= 检验"真懂 vs 蒙对"的灵敏探针。**

### 设计原则
1. 输入视作"同步/无序的视角集合"，信号只来自**跨视角空间结构**，不做时序分段、不假设相邻相似。
2. **pose-free / depth-free 优先**（实拍常无标定）。3. **training-free 优先**，必要时极小蒸馏头。4. **普适性即卖点**。

### 统一内核：一个原语，三种投影（方案脊柱）
**共享原语 = pose-free 跨视角对应**：用视觉特征匹配（互最近邻 / 区域 embedding 聚类）把"同一 3D 物体/区域在不同视角中的 token"关联起来，赋**跨视角全局 id** 并选**最佳视角代表**。它同时解决两目标：推理（补 All-Angles 的对应短板）+ 加速（按对应去重）。
- **Reason-only 投影 → R1**：用对应建全局关系图供推理，不动 token 数。
- **Compress-only 投影 → C1**：用对应去重 token，不显式建图。
- **Joint 投影 → U1 ⭐**：建推理结构的同一次计算顺带产出压缩掩码——既加速又提升（= 文档追问"这些信息能不能用来做多视角 token 压缩？"的落地）。

### 空白（带 arXiv，可写进 motivation）
- 跨相机（**非时序**）token 去重 ≈ 0 篇；同步多机位/ego-exo token 压缩 = 全空白（SeGPruner 2603.29437、Geo3DPruner 2604.18260 只做静态扫描+已知位姿深度；QTSplus 2511.11910 把 multi-camera 列 future work）。
- 几何 × query 双感知多视角选 token = 无人做。
- 视角数量/视角 distractor 对空间推理的隔离实验 = 基本没有。
- Set/Graph-of-Mark（2603.06663 / 2310.11441）只在单图，无跨视角版。

### 三个坑（反面证据）
- 别 naive 跨视角均值融合（Point3R/Cog3DMap mean-merge 失真）→ 用置信度/身份感知融合。
- 别走"生成想象视角"的 test-time scaling（ViSA 2512.05809：MMSI-Bench 上想象视角是信息瓶颈、verifier≈随机）→ 只在真实视角子集上做。

### 一、空间推理能力提升（training-free）

**⭐ R1：跨视角「局部→全局」结构化推理（Reason-only 投影）**
- *Challenge*：All-Angles 两大死穴=跨视角对应失败 + 相机位姿失败，且 IC 高（很多"答对"是蒙）。Graph/Set-of-Mark 只在单图建关系图，无跨视角统一机制。
- *Motivation*：**扎实动机=All-Angles 实测的跨视角对应失败 + 位姿失败 + IC 高（真·多视角直接测到）**。VSI-Bench 的"局部世界模型/缺全局"**仅作灵感、非前提**（2026-06-05 修正）——其轴是**空间距离非视角**（跨视角物体也可能距离近而判得准）、且为**单相机室内视频**，"真·多视角是否跨视角拼不起来"未验证，已列为 R1 前置验证实验（见 [Spatial-Reasoning.md]）。
- *方法*：① 逐视角 SoM 标号 + MLLM 输出该视角局部关系/属性；② pose-free 跨视角对应（区域特征互最近邻 + 外观/布局一致性）赋全局 id；③ 全局拼接成关系图（节点=全局物体 + 跨视角聚合属性；边=调和后空间关系；由对应反推粗相机视点排序——补位姿短板）；④ 紧凑全局图（文本）+ 标记图喂 MLLM。
- *新颖性*：Set/Graph-of-Mark 首个跨视角扩展；pose-free；把"局部→全局统一"做成 training-free 层。指标用 IC + Acc。

**R2：把「视角相关性剪枝」做成推理增强（idea#1）**
- *Challenge*："少而精"证据全在帧/时序；视角 distractor 对空间推理无人隔离；压缩→提精度未当多视角一等目标；注意力信号不可靠。
- *Motivation*：无关视角分散注意力当 distractor；只留 query 相关视角/token，应同时涨精度、降 token。
- *方法*：① 因果隔离实验（固定答案相关视角，加无关视角 / 关键视角排中间 → 测 Acc+IC，验证"视角版 lost-in-the-middle"）；② training-free 解法：用外部 grounding（REC-lite/开放词表检测，**不用** LLM 注意力）给每视角打"是否含被问物体"相关分 → 删 distractor、关键视角前置；③ 展示 Acc↑ 且 token↓。
- *新颖性*：首个隔离"视角 distractor 对空间推理的因果"，把视角/token 剪枝重定义为多视角精度工具。

**（可选）R3：真实视角子集上的空间自一致（idea#3，谨慎）**
- 不生成新视角（ViSA 已证失效），对真实视角的不同子集分别推理、按几何一致性聚合投票。VSI-Bench 说"整段 self-consistency 无效"，但"视角维度 self-consistency"是干净反直觉假设。风险：verifier/聚合准则难设，作增强模块不作主线。

### 二、推理加速 / Visual Token 压缩（training-free）

**⭐ C1：跨视角对应感知的 token 去重（Compress-only 投影，pose-free）— 加速首选 / 最大空白**
- *Challenge*：现有压缩把同步多视角当独立图（[CLS] per-image，同物体每视角各留一份）或当时序相邻帧（FastVID 2503.11187 / HoliTom 2505.21334 假设相邻相似，对同时刻不同视角无效、误伤互补视角）。Prune2Drive：各视角独立剪、不建模视角关系、ratio 靠验证集、只能定"裁多少"。SeGPruner/Geo3DPruner 要已知位姿+深度、只做静态扫描。
- *Motivation*：正确压缩单元是"跨视角的同一物体"。删跨视角重复 token、每物体留最佳视角代表 → 大幅减 token 且修复计数。
- *方法（pose-free）*：① 跨视角对应聚簇；② 每簇置信度/身份感知保留最佳代表（按正面度/清晰度/检测置信度），**不做均值融合**，视角敏感物体保 K 个代表；③ 显式保留单视角独占 token（保 scene completeness，补 Geo3DPruner 短板）；④ LLM 前完成。
- *新颖性*：首个面向同步多机位/ego-exo 的跨相机（非时序）去重；pose-free；以"物体跨视角"为单元；去重=提精度（计数）；逐条反驳 Prune2Drive 三宗罪。验证：All-Angles 计数 + EgoExoBench 2507.18342。

**C2：几何 × query 双感知的多视角选 token（idea#2）**
- *Challenge*：3D 多视角方法 query-agnostic；query 方法（CDPruner 2506.10967 / QTSplus）无几何无多视角；交叉点空白。Geo3DPruner 拿了深度/位姿却没用上。
- *Motivation*：空间 QA 本质问特定物体；留"既 query 相关、又跨视角几何多样/完整"的 token（Ego3D-VLM"只编码被问物体"+12%）。
- *方法*：query→被指物体 → token 打分 = query 相关度（对被指物体描述 cross-attn）× 跨视角条件多样性（DPP，仿 CDPruner）→ 选"被指物体最佳视角 + 最小上下文"。training-free。
- *新颖性*：首个几何×query 双感知多视角选 token；把 Geo3DPruner 浪费的几何用于 query 相关跨视角覆盖。

**C3：干掉几何骨干「白嫖」开销 →「真加速」**
- *Challenge*：Geo3DPruner/Cog3DMap/VG-LLM 每次跑重几何模型（VGGT-1B/Point3R），Geo3DPruner"加速很有限"正因 VGGT，且无人计入效率账。
- *Motivation*：跨视角对应的合并键只需粗对应，不需重建级几何 → 用 MLLM 自带 ViT 特征匹配（免费）或极小蒸馏头。
- *方法*：先做"仅 ViT 特征匹配 vs 用 VGGT"消融（证对应质量相当、成本骤降）→ 主线 training-free 视觉特征匹配；报告含几何开销的端到端 FLOPs/延迟/显存（无人报告的指标）。
- *新颖性*：首个量化并消除几何骨干开销，让多视角压缩真的快。

### 三、⭐⭐ 统一：既加速又提升（Joint 投影，真正卖点）

**U1：任务感知跨视角认知速写（Task-aware Cross-View Cognitive Sketch）**
- *Challenge*：Ego3D-VLM 把被问物体 3D 坐标抽成文本认知图用于推理，但是附加进输入（延迟 +31%）、没减视觉 token；Cog3DMap 靠"每 3D 位置唯一 token"压缩，却要训练 + 要几何模型 + 为视频递归设计。没人用"任务相关跨视角抽取"去反向裁原始视觉 token。
- *Motivation（关键洞察）*：构建"任务相关跨视角空间结构"这一动作本身就标定了哪些 token 重要（被问物体、最佳视角）、哪些冗余（重复视角、无关背景）→ 建推理脚手架的同一次计算顺带产出压缩掩码。压缩与推理增强合成一个操作：只留速写引用到的少量 token → token↓（加速）+ 任务相关信号更密、distractor 更少（推理↑、跨视角幻觉↓）。
- *方法（training-free）*：① query→被指物体；② pose-free 跨视角定位被指物体、留最佳视角 token，建紧凑跨视角认知速写（物体节点 + 跨视角融合位置 + 关系，文本/soft token）；③ 原始视觉 token 裁到 {被指物体 token + 最小几何锚点上下文}，删冗余视角与 distractor；④ {裁后 token + 速写} 喂 MLLM。
- *新颖性*：首个让"多视角空间推理表征"与"token 压缩掩码"成为同一个东西；逐条反驳 Prune2Drive（独立）/Geo3DPruner（VGGT 重）/Cog3DMap（要训练、为视频）全部缺陷；普适多视角图像。同时投 Ego3D + All-Angles。
- *与 R1/C1 关系*：三者共用同一原语——R1=只建图投影，C1=只去重投影，U1=两者合一。写论文=一个框架 + 三种配置。

### 落子顺序 + 评测设计

### 风险与回退
- 跨视角对应不准（无位姿、视角差大）→ 用 query 相关性兜底，只在高置信对应处去重。

- U1 的 REC 召回冗余（Ego3D-VLM 已知问题）→ 加 query 约束 + 置信度过滤。

## 竞品精读批次②：多视角压缩 / 视觉提示（2026-06-04）

> 6 篇与本课题强相关的论文精读。每篇含基本信息 / 方法要点（判定信号·training-free?·pose/depth?·单图or多视角or视频）/ 关键结果 / 与我们方案的切割。详尽差异已同步进 [Spatial-Reasoning.md] / [Visual-Compression.md] 的"新颖性与区分"。

### 精读 6：Coarse Correspondences（CVPR 2025）`arXiv:2408.00754`
**R1 的头号危险近邻。** training-free（闭源 GPT-4V/O；开源 LLaVA 需 120 万样本微调）、pose-free/depth-free。
- 方法：用**视频跟踪器 Tracking-Anything/SAMv2**抽实例级跨帧对应 → 选 top-K 共现物体 → 在质心叠**数字 ID 标记**喂 MLLM。**靠"同号=同物"传达对应，不画连线、不建关系图。**
- 关键结果：ScanQA CIDEr 72.2→87.0(+15)、OpenEQA 49.4→59.1、EgoSchema +6.0、R2R SR +11；缓解相机运动偏置(+17.3)。
- 关键定性：输入"不必是视频，也可多视角图像集"，**但对应机制本质是时序 tracking（依赖帧间连续性），实测全是时序场景，从未在 All-Angles/Ego3D-Bench 等同时多相机基准测试。**
- **与 R1 切割**：(i) 它**不建关系图**（R1 核心立点）；(ii) 它靠**视频 tracking**、处理不了大基线同时多相机；(iii) 未测目标基准。R1 必须诚实承认"叠标记+对应+喂MLLM+training-free+pose-free"同源，新颖性**只能压在 关系图 + 静态多相机 + 非tracking对应** 三点。

### 精读 7：SeGPruner（短文，上海大学）`arXiv:2603.29437` （重要，与multi-view强相关，但方法貌似与Prune2Drive相似，重要性+多样性token选择）
**C1 的直接竞品（几何路线）。** training-free，但**硬依赖 GT 深度 + GT 相机位姿**。
- 方法：插在视觉编码器与 LLM 间，两阶段：STS（视觉编码器 self-attention 显著性，**query-agnostic**）选重要 token + GTD（"特征相似度 + 3D 欧氏距离"融合距离的 **FPS** 最远点采样）补空间多样性。**把 token 用 depth+pose 反投影到统一世界系**做几何比较。**非物体 re-id，是 patch 级几何 FPS。**
- 关键结果：ScanQA 仅留 23% token > 全量；token 降 91%、延迟降 86%；9% 保留仍 95.3% 性能。只测 ScanQA/OpenEQA（ScanNet 系，自带 GT 深度位姿）。
- **与 C1 切割**：C1 **pose/depth-free**（最硬 delta）；C1 **物体级 re-id + 每物体最佳代表** vs 它 **patch 级几何 FPS**。training-free 两者相同（不能当卖点）。

### 精读 8：Merge3D（CVPR 2026，NUS/PolyU）`CVF，arXiv 待补`
**C1 直接竞品（merging 路线）+ 给出关键负面证据。** 建于 VG-LLM 双编码器（2D + VGGT），**pose-free（VGGT RGB-only 隐式几何）但需训练**（训 SemGeo Merger，8×H100 仅 2–3h）。
- 方法：SemGeo Merger——2D self-attention 选 top-K 主导 token；上下文 token 按 **2D语义×3D几何相似度（乘法门控）** 指派给主导 token；**合并用算术平均（Eq.6）→ 有均值失真风险**（靠"簇内同质"间接缓解）。**无物体 re-id、无 query 感知（列为 future work）。**
- 关键结果：减 70% token、~3× 加速；5% token 时全面超 VisionZip-2D/3D/Random。**BLINK Multi-View 子集卡死 55.6%（≈基线），作者承认"大视角变化需 token merging 之外的机制"** ← **C1 的最佳战场**。
- **与 C1 切割**：C1 = 完全 training-free + 显式跨视角同物体 re-id（攻其 Multi-View 短板）+ query 感知 + 最佳代表（非均值）。

### 精读 9：Seeing Once Is Enough?（Workshop @ ICLR 2026，NTHU）`OpenReview，无arXiv` （重要，看它的多帧体素是怎么构建的）
**C1 竞品（online 几何路线）。** training-free，但**硬依赖 posed RGB-D（深度+位姿）**。
- 方法：流式逐帧，深度+位姿反投影→**体素化**→维护全局已访问体素集 S→当前帧落入已观测体素的 token 判为冗余剪掉（**纯几何重叠率判据，非视觉相似**）。"看过一次就够"。
- 关键结果：ScanQA/SQA3D/OpenEQA token 降~50% 且**涨点**（缓解 attention dilution）；τo=100% 最优。**只报 token 数，无 FLOPs/延迟。**
- **与 C1 切割（很干净）**：判据 **几何坐标对齐 vs C1 纯视觉匹配**；输入 **posed RGB-D 流 vs C1 pose-free 稀疏多视角图**——在 All-Angles/Ego3D-Bench（无位姿深度）上**它直接不可运行**。两维度正交。其"去重→涨点"是 C1 的有利论据。

### 精读 10：Proxy3D（CVPR 2026，清华/Berkeley/Panasonic）`arXiv:2605.08064`
**C1 竞品（语义聚类路线）+ 削弱"避免均值"卖点。** pose-free/无传感器深度（几何靠 VGGT），但**重训练**（4 阶段 SFT、62 GPU-h、自建 318K SpaceSpan）。
- 方法：SAM2 语义 label 分组 → 组内按 VGGT point-map 坐标 **KNN 聚类** → proxy = **FPS/KNN 采样的代表点+邻域特征（不是均值！）**；BFS 序列化 + 3D 位置嵌入 + identifier/semantic 符号嵌入。一个物体→**多个 proxy**。
- 关键结果：700 token（vs correspondence 法 8000、Spatial-MLLM 3096）；VG 任务 SOTA（Uni@0.5=84.0）；VSI-Bench 47.0（开源#2）；Scan2Cap 明显弱（简洁vs语义 tradeoff）。
- **与 C1 切割**：①**training-free vs 重训练**（最硬）；②**实例级跨视角 re-id vs 语义类内几何采样**（它一物体多 proxy、不做身份合并）；③**单一最佳代表优选 vs 几何均匀采样**。⚠️ **重要修正：Proxy3D 也不用均值融合（用 FPS/KNN 采样），所以"我们避免均值失真"不能当主要卖点**——C1 真正 delta 是 training-free + 实例 re-id + 最佳代表选择准则。

### 精读 11：ViKey（CVPR 2026，延世大学）`arXiv:2603.23186`
**R1 仅相关（同 visual-prompting 母题），非竞品（任务正交：时序 vs 空间）。** training-free、plug-and-play。
