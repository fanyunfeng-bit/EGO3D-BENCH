# CVSP — 故事 / Motivation(工作版，2026-06-17）

> 本文件只装"论文怎么讲"的故事线 / motivation。**方法规格（定型版）见 [CVSP-Method.md]**；实测数据、竞品切割见 [Visual-Compression.md]（§实测发现）。
> 工作名 CVSP = Cross-View（把 VisPruner 的逐视角选择升级到跨视角）。
>
> **⚠️ 实验现状（2026-06-18，必读）**:本文 §1–7 + 设计日志是 motivation/方法的**演进记录**,带着"压缩**同时提精度**"的乐观假设。但截至 2026-06-18 的实测(见 [Visual-Compression.md] §D–J)**已推翻精度部分**——query-agnostic 的所有选择器(显著性 / 多样性 / leverage / 几何锚点 / log-det 引擎)都**没有稳定赢过 stratified random**。读本文时,凡"提精度"一律按"**待验证、大概率不成立**"理解。最新裁定见文末「实验现状」。文中"P4 进行中""下一步=跑体检"等是写作当时的进度,**均已完成**,以文末为准。

---

> **一句话**:为单图设计的视觉 token 剪枝在多视角上失灵——连随机删都赢不过;根因是"聪明选"的两条轴(显著性、多样性)是逐视角的，没做跨视角关联。我们把它们升级到跨视角，training-free，同时省算力、提精度。

## 1. 问题
多视角 VLM(环视驾驶、室内多帧、ego-exo)越来越多。视觉 token 随视角数线性涨、注意力按平方涨 → 真实计算负担(prefill FLOPs / KV 随视角数恶化)。压缩是自然解法，但现有方法都**为单张图设计**。

## 2. 我们的发现(实证)
- **发现一:多视角 token 高度冗余，而且多余 token 在帮倒忙。** 随机删 75–90%，精度几乎不掉、**有时更高** → 多余 token 在稀释注意力、干扰模型 → 压缩能**同时省算力和提精度**。
- **发现二(反常):为单图设计的 informed 剪枝(VisPruner)拿不到对随机的任何优势，有时更差。** → **问题出在"单视角地聪明选"这个范式本身**(在 InternVL3-8B 和 Qwen2.5-VL-7B 上都如此，排除单模型偶然)。

## 3. 诊断:单视角范式为什么不适配多视角
VisPruner 这类 = **逐视角**做两件事:留显著(显著性轴)+ 留多样(多样性轴)。问题全在"逐视角"——它让"同一块显著内容在多视角各留一份"，带来两个后果:
1. **有用内容被重复留 N 份**:预算被同一物体吃掉，真正互补的(单视角才看得到的)反而没预算留;
2. **无关的显著内容在多视角重复出现、被过度保留**，污染保留集;
3. **根上的原因:两条轴都没有跨视角关联**——既认不出"同一内容跨视角"，也做不到跨视角去重与覆盖。

## 4. 核心洞察
> **显著性和多样性这两条轴，都必须从逐视角升级到跨视角。** CVSP 不发明新轴，而是把 VisPruner 已有的两条轴**跨视角化**。

## 5. 方法 CVSP(两件事，各对一个后果，training-free)
- **跨视角冗余折扣**:一块内容在越多视角被重复保留，其重要性折得越狠，只留最佳一份。**不区分有用/无关——只看重复了几次**，因此同时治"有用内容的浪费重复"和"无关显著内容的多视角放大"。
- **全局覆盖保独有**:单视角独有的内容(无跨视角副本)保底留一个代表，不被当冗余删掉;对单视角内容**一视同仁**(query-agnostic 下无法安全区分有用与噪声，强行区分会误删有用的独有内容)。
- *(实现:用一套廉价的跨视角特征匹配来判"重复几次"——**此匹配为暂定方案，有效性待验证**;不引入额外大模型、不训练。)*

> 设计史(防止反复):曾考虑第三个组件"跨视角旁证去假显著"，**已删**——它要压低单视角才显著的内容，与"全局覆盖保独有"直接矛盾、且易误删有用独有内容。无关显著的噪声，凡是被多视角放大的都由"冗余折扣"一并治掉;只在单视角出现的那点残留噪声，query-agnostic 下不安全去除，留一个代表即可、危害有限。

## 6. 诚实的边界(主动写进去)
CVSP 是 **query-agnostic** 的:能治"跨视角重复"(含被多视角放大的无关显著)。**治不了**"只在单视角出现、又恰好无关"的显著物——它和"单视角独有的有用内容"在 query-agnostic 下无法安全区分，只能一视同仁留一个代表(危害限在一个 token)。彻底去这类要靠 query-aware 版(后续工作)，本文不碰，以保持简单、可预计算、可复用。

## 7. 验证计划(决定成败)
- **核心门槛:在去重/覆盖敏感的任务上，CVSP 要稳定 > 随机 且 > 逐视角 VisPruner**(发现二说明赢过随机是硬门槛，**选对任务比方法本身更关键**)。
- **前提自检(防"模型没看图"的攻击)**:在目标任务上验证模型确实依赖视觉内容——换黑图/噪声 → 精度掉向随机——所以 token 选得好不好在这些任务上真的有意义。**(只说"视觉有用"，不量化它占比。)**
- **任务选择**:打**视觉扛大头**的任务(物体距离、counting 等)，而非被语言先验主导的任务。
- **广度**:跨 InternVL3 + Qwen 两架构，跨 VSI / Ego3D(后续 All-Angles)。
- **效率**:报含估计开销的端到端 FLOPs / KV / 延迟，不只 token 数。
- **关键消融**:逐视角 vs 全局;两个组件逐个开关;**跨视角匹配方案本身的有效性**(暂定，要先验证)。
- **补证(P1–P3 已完成 2026-06-17,见 [Visual-Compression.md] §实测发现 G)**:VisPruner 留的 token 比 random **更跨视角冗余、覆盖更差**——4 个 ACC 任务、每个样本 100%、配对 t>45。诊断已成硬证据;"informed 赢不过 random"的根因 = random 保留集冗余更低。**P4(因果,进行中)**:全局 FPS 显式压低冗余,看能否精度真超 random(同预算)。

---

**贡献三句话**:① 反直觉发现+诊断——单图剪枝范式在多视角上连随机都赢不过，根因是两条选择轴没跨视角化;② 简单 training-free 的对症方法——把两条轴升级到跨视角;③ 同时改善效率与精度，跨架构、跨 benchmark 验证。

**当前最大风险**:informed ≈ random，方法可能赢不过 random。motivation 再漂亮，方法不赢就没文章 → 下一步先验证"能在某类任务上稳定赢过 random + VisPruner"，再谈其他。

## 方法设计日志（2026-06-17）：从"压冗余"到"锚点信号"

**P4 因果实验否定了"压冗余"路线**(数据见 [Visual-Compression.md] §G/P4):全局 FPS 把跨视角冗余 R 压到最低(0.44 << random 0.55 << visp 0.62),但精度**没超过 random**(Object 0.404 vs 0.412、Ego 0.468=0.468)。修正:
- **"R 低 → Acc 高"是必要非充分,不是优化目标**。FPS 用"挑特征离群点"这种烂办法压低 R → 低 R 没换来高 Acc。**别再优化 R 这个代理**(守此方法论)。
- 真问题 = **"什么 query-agnostic 信号能选出比均匀 random 更有用的 token"**。已知两条死路:视觉显著性(VisPruner)、特征多样性(FPS),都败给 random——因为它们是"**通用**重要性"而非"**空间推理**重要性"。

**Leading 候选:几何承重 / 跨视角锚点。** 对空间任务,"有用" = 撑起全局坐标系的"跨视角一致 + 结构独特"的点(角/地标),不是显眼也不是离群。query-agnostic(坐标系跨任务共需,可预计算)、无 VGGT。
- **锚点分(max 版,跨重叠普适)**:
  `anchor(t) = max_{u≠v} [s₁(u) − s₂(u)]₊ · s₁(u)`
  s₁/s₂ = t 在视角 u 内的最/次相似;`[s₁−s₂]` = Lowe margin(独特性,一并滤掉重复纹理 + 同类不同物);`·s₁` = 匹配强度加权(防"弱匹配假独特")。**用 max 不用 Σ**:2 个视角即可三角化一个 3D 点,Σ 会奖励"被更多视角看到"、系统性埋没低重叠(驾驶/Ego)数据里的 2-view 真锚点。s₂ 取离 s₁ 有空间距离的次近邻。
- **覆盖兜底**:anchor≈0 的单视角独有内容(互补信息)靠覆盖留;探针里 coverage 直接取**均匀 random**。
- **盲区(诚实)**:此方案只做"坐标系骨架",**没显式管"前景被问物体"(显著性)**——先看 anchor+coverage 够不够,不够再把显著性当**组合项**加(它单用已败,不单押)。"问哪个物体"本质 query-相关 → query-aware v2。

**最干净的验证 = 单变量因果探针**:coverage=均匀 random 当无偏底座 → **注入 k 个(去重的)anchor token 替换 k 个 random** → 扫 k=0→budget 的精度曲线。k=0 即纯 random;精度随 k 升 = anchor 对空间理解重要;平 = 信号死。
- 跨数据集量一个诚实指标:每个数据集有多少 token 拿得到非零 anchor 分(低重叠 → 稀疏 → 该信号天生高重叠 VSI 给得多、低重叠驾驶给得少)。

## 整合 + 决策（2026-06-17 续）：CVSP 成熟版 = 体检 → 地标分 → 招考察队

**跨领域头脑风暴的整合**(5 个研究 agent 跨 coreset/子模、最优实验设计、LLM 文本压缩、SLAM 关键点选择、压缩感知/采样理论,**独立收敛到同一配方**):新 CVSP 是 VisPruner→多视角这条线**想透后的成熟版**,三件套:
- **招考察队(选择引擎)**:一步挑出"又重要、又不重复"的 token(质量加权 log-det 贪心:质量分=重要,相似度项=去重)。取代 VisPruner 的"先挑显著 top-k、再补多样性"两步、逐视角。
- **地标分(重要分)**:`角不角 × 几个视角看到 × 对得清(跨视角独特)`,取代显著性。**"角不角"暂定用方法 B**(特征 vs 同视角邻居的差异;省事、贴模型),方法 A(原图梯度 Shi-Tomasi)留作对照。
- **体检(总闸)**:动手前先测 token 池有没有"重点"(少数块鹤立鸡群 vs 块块差不多)。平 → random 不可超,pivot;尖 → 继续 + 看"鹤立鸡群"的是不是正好地标。

> 老 CVSP 的"跨视角冗余折扣"内化进招队的"不重复"规矩;"全局覆盖"变覆盖兜底;灵魂换成"地标分"(把"重要"从显眼改成空间地标)。

**显著性决策(已认可)**:**v1 先纯锚点**(最干净,先看能否超 stratified random);若实测"漏前景物体"是丢分主因 → 把**显著性当第二重要分加回**(锚点为主、显著为辅)。不是永久砍——**新引擎的全局去重正好治好了显著性当年"逐视角留一堆重复"的病**,给它一次改过自新的机会;要不要由消融定。

**已有同类工作**(facility-location/FLoC 2511.00141、quality×diversity DPP/CDPruner 2506.10967、leverage/SVD-Prune 2604.11530)→ 新颖性**必须押在**:多视角 3D 空间 regime + 几何地标质量分 + "在 random 击败显著性与多样性的地方我们超过了 random"的严谨故事。baseline 用 **stratified random**(逐视角逐区域配额,Window-FastV 2502.11501),非 iid random。

**下一步 = 跑体检**(Ego3D + VSI 双数据集)。

## 实验现状（2026-06-18，全局裁定）

方法演进链:压冗余(P4) → 锚点(max-Lowe) → 体检 → 四方对照 → 实验 A → combo(进行中)。逐项裁定(数据全在 [Visual-Compression.md] §G–J):

- **P4(全局 FPS,§G/P4)**:把跨视角冗余 R 压到最低,精度**没超过 random** → "R 低→Acc 高"必要非充分,**别优化 R**。
- **体检(§H)**:token 池有效秩 ~84 ≪ 1536(高度冗余、且跨视角),但只是"**温和的尖**"(油水小);**keep10/25 预算 > 有效秩 → 那里谁都赢不了 random**,真正战场在 **keep ≤ 5%**。地标 ≠ 高 leverage 块(相关≈0)。
- **四方对照(§I)+ 实验 A(§J)**:keep5,stratified random vs anchor / leverage / engine。**anchor 净 ≈ 0**(空间任务多为小正 +1~3pp,但**时序任务 −7pp**、route −3pp);leverage 最差;engine 不胜 plain anchor。→ **没有 query-agnostic informed 方法稳定赢随机。**
- **combo(已取消,2026-06-18)**:原计划 `q = norm(anchor)+norm(saliency)` 喂 engine 跑对照。用户拍板**不再跑探索性 combo**,直接把方法**定型为配额式三档 anchor+saliency+coverage**(规格见 [CVSP-Method.md])。理由:加权和是固定盲配比、易塌向一边;改用**三档配额各自选 + 每视角保底**更鲁棒,且把挑战(1)(3)合并、saliency 重定位为"前景下限"。
  - **精度赌注改押极端压缩区**(keep≤5%,预算 < 有效秩 ~84)——体检结论说那里选择质量才起作用,是验证留下的唯一缺口。主曲线 `ACC vs r∈{25,10,5,3}%`,假设 0.05/0.03 处 CVSP 与 random 分开。
  - 成败门槛(预注册,见 [CVSP-Method.md] §8):`r≤0.05` 多数任务明显 > stratified random 且 `r≥0.10` 不退化 → 精度故事成立;否则 → **回去迭代方法(调 ρ/信号、或 query-aware v2)**。**不走"效率/发现型论文"退路(2026-06-18 决策)**——反直觉冗余发现只作 CVSP 的 motivation,不单独成文。

**一句话现状**:CVSP 已**定型为三档(anchor+saliency+coverage)配额式跨视角选择器**([CVSP-Method.md])。作为"精度方法"仍**待 §8 主曲线验证**(赌注在极端压缩区)。**已撤掉"效率/发现型论文"退路**——反直觉冗余发现只当 motivation;**没了安全网,项目能否出精度论文完全押在 CVSP 真能赢 random**(诚实提示)。query-aware 是唯一没碰的精度杠杆,留 future work。
