# Algorithm Run Guide (Visual-Token Compression)

This guide is for **running and reproducing the visual-token-compression experiments** added on
top of the Ego3D-Bench harness. It tells you, for each method, **which files are involved, the
exact command, what every parameter means, where the output lands, and how it is scored.**

> Scope of this document: the **compression-research layer** (our additions) + VSI-Bench + the
> diagnostic scripts. It does **not** re-document the original Ego3D-Bench benchmark runners
> (`models/qwen2.5_vl.py`, `models/internvl3.py`, `*_ego3dvlm.py`) — for those see `RUN.md` and
> `RESULTS_baseline.md` at the repo root.

---

## 0. Environment & global conventions

**Run everything from the repo root.** All Python must be the dedicated `ego3d` conda env
(`transformers==4.51.1`, hard-pinned — other versions break the runners). See `requirements.txt`.

```bash
# Option A: activate the env （创建你自己的环境）
conda activate ego3d

# Option B (what the run_*.sh scripts use): prefix PATH
export PATH=/home/fyf/miniconda3/envs/ego3d/bin:$PATH
```

Environment variables every run needs (the `run_*.sh` wrappers set these for you):

```bash
export PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True   # avoids fragmentation OOM on a 24 GB GPU
export HF_HUB_OFFLINE=1                                    # all weights + datasets are cached locally
```

Global facts that apply to **every** runner below:

- **Model:** `OpenGVLab/InternVL3-8B` (InternViT-300M vision encoder + Qwen2.5-7B LLM, 28 layers).
  Resolved from the local HF cache; do not change `--model_path` unless you have another checkpoint.
- **Per-tile token granularity:** InternVL3 produces **256 tokens per image tile** after
  pixel-shuffle — that is the unit the compressors select over. `keep_ratio = 0.1` ("keep10") keeps
  ~26 tokens/view; `0.05` ("keep5") ~13.
- **Resume / determinism:** every runner is **resumable by JSONL line count** — it counts existing
  rows in the output file and skips that many samples (`<=`, so an interrupted sample is retried,
  not duplicated). Sample order is deterministic, so an `--n 200` file is a true prefix of the
  `--n 99999` (full) file and will be **auto-extended**, not re-run. Random methods are seeded
  per-(sample, view), so reruns reproduce identical pruning.
- **Long runs:** a full-data sweep is many hours. Launch detached so it survives terminal close:
  ```bash
  setsid nohup bash scripts/run_ego3d_full.sh >/dev/null 2>&1 &
  ```
- **Scoring caveat (important):** on Ego3D **score by ACC, not RMSE.** Pruning *artifactually*
  improves RMSE (regression to the mean / fewer clamped outliers) while ACC drops — see §6.

| `utils/{cam_info,common,eval,internvl3_utils}.py` | upstream | dataset/eval/image helpers (shared by all) |

**The single most important distinction:** there are **two harnesses** and our headline methods
live in only one of them.

- **Harness A** (`models/internvl3_compress.py` & the two `*_vsibench.py`) is selected with
  `--compress_method`, which is limited to the **registry**: `none / vispruner / random`. It is the
  **efficiency + simple-baseline** harness (it writes `*.result.json` with FLOPs/KV/mem/time).
- **Harness B** (`scripts/cvsp_curve.py`, `scripts/qstage_curve.py`) is where **a20s40,
  block-cvsp, and the two-stage method** live. You select them with `--methods` / `--signal`,
  **not** with `--compress_method`. It writes accuracy-only JSONL to `logs/cvsp/`.

## 2. Data preparation

**Ego3D-Bench images** are expected under `Ego3D-Bench/images/` (the default `--image_root`). The
QA itself loads from the HF dataset `vbdai/Ego3D-Bench['test']` (cached locally).

**VSI-Bench** must be prepped once before any VSI run:

```bash
# quick (one source, ~1 GB cached):
python scripts/prep_vsibench.py --sources scannetpp
# full (~5.3 GB):
python scripts/prep_vsibench.py --sources scannet,scannetpp,arkitscenes
```

`prep_vsibench.py` parameters:

| Flag | Default | Meaning |
|---|---|---|
| `--sources` | all 3 | comma-sep subset of `scannet,scannetpp,arkitscenes` |
| `--frames` | `6` | frames uniformly sampled per video (these become the "multi-view" input). **Must match the `--frames` you later pass to the VSI runners.** |
| `--max-scenes` | `0` (all) | cap distinct scenes per source (for a quick subset) |
| `--out` | `data/vsibench/vsibench_items.json` | item manifest consumed by every VSI runner |

Output: frames in `data/vsibench/frames/<source>__<scene>/v{0..k-1}.jpg` and the manifest
`data/vsibench/vsibench_items.json` (all 10 VSI question types).

## 3. Running the methods

For Harness B (cvsp_curve / qstage_curve) a "config" is a `dataset:task` pair, e.g.
`ego3d:Localization` or `vsi:object_rel_distance`. The two Ego3D number tasks
(`Ego_Centric_Absolute_Distance`, `Object_Centric_Absolute_Distance`) are auto-detected and scored
by **RMSE**; everything else is **ACC**.

The 7 Ego3D spatial tasks used in the full benchmark (reused as a string in the drivers):

```
ego3d:Ego_Centric_Absolute_Distance              (RMSE)
ego3d:Ego_Centric_Absolute_Distance_MultiChoice  (ACC)
ego3d:Ego_Centric_Relative_Distance              (ACC)
ego3d:Localization                               (ACC)
ego3d:Object_Centric_Absolute_Distance           (RMSE)
ego3d:Object_Centric_Absolute_Distance_MultiChoice (ACC)
ego3d:Object_Centric_Relative_Distance           (ACC)
```

### 3.1 Baselines / controls — FULL, plain_random, vispruner (Harness B)

These three are the comparison set for every method. `baseline` = no compression (all tokens);
`plain_random` = uniform random over the global token pool at the same budget; `vispruner` =
per-view VisPruner.

```bash
python scripts/cvsp_curve.py \
  --methods baseline,plain_random,vispruner \
  --ratios 0.1,0.05 \
  --n 99999 \
  --configs "ego3d:Localization,ego3d:Object_Centric_Relative_Distance"
```

Output: `logs/cvsp/<ds>.<task>.baseline.jsonl` and
`logs/cvsp/<ds>.<task>.keep<pct>.<method>.jsonl`. An ACC/RMSE table prints when each config
finishes.

> Note: `strat_random` (per-view 2×2-block uniform) and `nuwa` (nuwa-lite) are also available in
> `--methods` as extra baselines; they were used in earlier studies and are not part of the headline
> comparison.

### 3.2 block-cvsp — single-stage, block-stratified (Harness B)

Block-CVSP = Layer-1 anchors (top `rho_a·K` by `cornerness × cross-view Lowe`, **no dedup** to
keep cross-view pairs) + Layer-2 block-stratified saliency with cross-view dedup. See
`Notes/CVSP-Method.md §12`.

```bash
python scripts/cvsp_curve.py \
  --methods block_cvsp \
  --ratios 0.1,0.05 \
  --rho_a 0.2 --brows 4 --bcols 4 --tau 0.85 \
  --tag=-bc4-a20 \
  --n 99999 \
  --configs "ego3d:Localization"
```

Key params: `--rho_a` anchor fraction (`0` = no-anchor ablation), `--brows/--bcols` the per-view
spatial block grid (4×4=16 blocks fits the keep10 budget better than 2×2; 8×8 was the earlier fine
grid), `--tau` cross-view dedup cosine threshold. **`--tag` is required to keep variants apart** —
the same tag is used both to write and to score the file (`...keep<pct>.block_cvsp<tag>.jsonl`).

### 3.3 a20s40 — single-stage 3-bucket CVSP (Harness B) ★ best single-stage

a20s40 is the `cvsp` method (three buckets: anchor / saliency / coverage + per-view floor +
facility-location coverage) tuned to **ρ_a=0.20, ρ_s=0.40** (hence "a20 s40"), with anchor reach
`κ=2`. This is the strongest single-stage informed selector found.

```bash
python scripts/cvsp_curve.py \
  --methods cvsp \
  --rho_a 0.2 --rho_s 0.4 --kappa 2 --tau 0.85 \
  --tag=-a20s40-k2 \
  --ratios 0.1 \
  --n 99999 \
  --configs "<the 7-task string from above>"
```

Param meanings specific to `cvsp`:

| Flag | a20s40 value | Meaning |
|---|---|---|
| `--rho_a` | `0.2` | anchor-bucket budget fraction `B_a = ρ_a·K` |
| `--rho_s` | `0.4` | saliency-bucket fraction `B_s = ρ_s·K` (coverage gets `1−ρ_a−ρ_s`) |
| `--kappa` | `2` | anchor reach: only the top `⌈κ·B_a⌉` tokens by raw cross-view support `L` are anchor-eligible; unfilled anchor budget flows to coverage. `0` = off |
| `--tau` | `0.85` | cross-view dedup cosine threshold |
| `--phi` | `1` (default) | per-view floor (min tokens guaranteed per view in the coverage stage) |
| `--tag` | `-a20s40-k2` | output-file suffix; **must match between run and score** |

Output: `logs/cvsp/<ds>.<task>.keep<pct>.cvsp-a20s40-k2.jsonl`.

### 3.4 Two-stage query-aware (Harness B) — `qstage_curve.py`

Stage 1 over-selects `N1 = r·N2` tokens with block-cvsp (4×4, ρ_a=0.2); Stage 2 prunes inside the
LLM at layer `K=14` (= L/2) down to `N2` by **query relevance**. The reported budget is the
layer-average `(N1·K + N2·(L−K))/L`, which equals the nominal `keep` ratio. See
`Notes/CVSP-Method.md §13`. Headline config = `--signal input_cos --r 7`.

```bash
python scripts/qstage_curve.py \
  --signal input_cos --r 7 \
  --ratios 0.1,0.05 \
  --n 99999 \
  --configs "<the 7-task string>"
```

Params:

| Flag | Default / headline | Meaning |
|---|---|---|
| `--signal` | `input_cos` | query signal for stage-2. `input_cos` = **paper-exact**, `cos(proj(vᵢ), q̄)` with `q̄ = avgpool(question token input-embeds)`, keep-set decided **pre-LLM** (FlashAttention-preserving — recommended). `cosine` = in-LLM layer-14 hidden-state cosine. `attn` = last-text-token→vision attention at layer K−1 (forces eager on that layer). |
| `--r` | `7` | over-select ratio `N1/N2`. `r=1` = single-stage, **no stage-2 prune** (equals plain block-cvsp at budget T — the no-prune baseline). `7` = Nüwa default. |
| `--K` | `14` | LLM layer at which stage-2 prunes (L/2 for the 28-layer LLM) |
| `--rho_a` | `0.2` | stage-1 block-cvsp anchor fraction |
| `--brows/--bcols` | `4 / 4` | stage-1 block grid |
| `--tau` | `0.85` | stage-1 cross-view dedup |
| `--max_new` | `48` | **vestigial** — generation uses `max_new_tokens=1024` internally; this flag is ignored |

Output: `logs/cvsp/<ds>.<task>.keep<pct>.qstage-<signal>-r<r>.jsonl`.

### 3.5 VSI-Bench

Two ways to run on VSI:

1. **Harness B (our methods):** pass `vsi:<task>` configs to `cvsp_curve.py` / `qstage_curve.py`
   (e.g. `--configs "vsi:object_rel_distance,vsi:object_rel_direction_hard"`). Requires
   `data/vsibench/vsibench_items.json` from §2. Same flags as Ego3D.

2. **Harness A (baselines + efficiency):** the dedicated VSI runners.

   ```bash
   # InternVL3 on VSI:
   python models/internvl3_vsibench.py \
     --compress_method vispruner --keep_ratio 0.25 --frames 6 \
     --category object_rel_direction_easy

   # Qwen2.5-VL on VSI:
   python models/qwen2.5_vl_vsibench.py \
     --compress_method none --frames 6 --category route_planning --attn flash_attention_2
   ```

   VSI Harness-A params: `--category` (`all`, one type, or comma-sep), `--compress_method
   {none,vispruner,random}`, `--keep_ratio`, `--important_ratio` (VisPruner important:diverse
   split), `--frames` (must match prep), `--limit` (smoke), and Qwen adds `--attn
   {flash_attention_2|sdpa|eager}`. Output: `logs/<model_name>-<tag>-vsibench/<type>.{jsonl,result.json}`.

### 3.6 Ego3D via Harness A (baselines + efficiency only)

If you specifically want the **efficiency numbers** (FLOPs / KV / peak mem / CUDA time) for the
simple baselines on Ego3D, use `internvl3_compress.py`:

```bash
# smoke (20 samples):
bash scripts/internvl3_compress.sh --compress_method vispruner --limit 20
# one full category at keep10:
python models/internvl3_compress.py --compress_method random --keep_ratio 0.10 \
  --category Object_Centric_Absolute_Distance_MultiChoice
```

Params: `--category` (one Ego3D category), `--compress_method {none,vispruner,random}`,
`--keep_ratio`, `--important_ratio`, `--limit`. Output:
`logs/<model_name>-<tag>/<category>.{jsonl,result.json}` where `tag` = `baseline` or
`<method>-keep<pct>`. **a20s40 / block-cvsp / two-stage are not available here** — use Harness B.

## 4. Full parameter reference (Harness B)

These are shared by `cvsp_curve.py` and (the relevant subset) `qstage_curve.py`:

| Flag | Default | Meaning |
|---|---|---|
| `--model_path` | `OpenGVLab/InternVL3-8B` | HF model id / local path |
| `--ratios` | `0.25,0.1,0.05,0.03` (cvsp) / `0.1,0.05` (qstage) | comma-sep keep ratios (per-view token fraction; budget `K = round(n_tok·ratio)·n_views`) |
| `--methods` | all | comma-sep subset of `baseline,plain_random,strat_random,vispruner,cvsp,nuwa,block_cvsp` (cvsp_curve only) |
| `--configs` | built-in | comma-sep `ds:task` (`ds ∈ {ego3d, vsi}`) |
| `--n` | `200` | samples per task (`99999` ≈ full) |
| `--seed` | `42` | base seed for random methods (per-sample/view derived) |
| `--image_root` | `Ego3D-Bench/images` | Ego3D image dir |
| `--vsi_items` | `data/vsibench/vsibench_items.json` | VSI manifest |
| `--tag` | `""` | suffix for `cvsp`/`block_cvsp` output files (variant separation) |

`--rho_a, --rho_s, --kappa, --tau, --phi, --brows, --bcols, --nuwa_lam, --nuwa_dist, --merge,
--merge_dist` are defined in §3.2–3.4. **Leave `--merge none`** — the saliency-merge variant was
tried and **failed** (`Notes/CVSP-Method.md §12.9`).

qstage-only: `--signal, --r, --K` (§3.4).

## 5. One-command reproduction (drivers)

The `scripts/run_*.sh` wrappers set the env vars, the 7/5-task strings, and chain the runs. They
are detached-friendly and resumable.

| Driver | What it runs |
|---|---|
| `scripts/run_ego3d_full.sh` | **Main Ego3D benchmark, FULL data, keep10/5:** baseline + plain_random + vispruner (cvsp_curve) **and** two-stage `input_cos r7` (qstage_curve), 7 spatial tasks |
| `scripts/run_a20s40_full.sh` | a20s40 (cvsp ρ_a=0.2/ρ_s=0.4/κ=2), FULL data, keep10, 7 tasks |
| `scripts/run_qstage_inputcos.sh` | two-stage paper-exact `input_cos`, r∈{3,7}, keep10/5, n=200, 5-task set |
| `scripts/run_qstage_cos.sh` / `run_qstage.sh` | two-stage `cosine` / `attn` signal sweeps (earlier studies) |
| `scripts/run_blockcvsp.sh` | block-cvsp anchor vs no-anchor ablation + nuwa-lite (n=200) |
| `scripts/run_ego3d_keep10.sh` | Harness-A vispruner+random over 6 Ego3D categories at keep10 (efficiency) |

Typical detached launch + monitoring:

```bash
setsid nohup bash scripts/run_ego3d_full.sh >/dev/null 2>&1 &
tail -f logs/cvsp/ego3d_full.log         # progress; per-task tables print as configs finish
nvidia-smi                               # ~16 GB resident while running
```

## 6. Scoring & how results are organized

- **Multiple-choice (ACC):** exact lowercase match of the `<answer>…</answer>` letter vs GT.
- **Number tasks (RMSE):** first regex number in `<answer>`, **clamped to 100 m**, RMSE vs GT.
  Only the two `*_Absolute_Distance` (non-MC) categories.
- **Always read ACC for Ego3D.** Compression makes RMSE *look* better (regression to the mean /
  fewer clamped outliers) even as accuracy falls — RMSE is misleading here.
- **Where numbers live:**
  - Harness B → `logs/cvsp/<ds>.<task>.<segment>.jsonl` (one row per sample:
    `{Processed_Pred, GT, n_kept}`). Re-score any time by re-running the same command (it just
    reprints the table over existing files) — nothing is recomputed for already-written rows.
  - Harness A → `logs/<model>-<tag>[-vsibench]/<category>.result.json` (metric **and** efficiency).
- Statistical reality: at `n=200`, 1 SE ≈ 3 pp; at full data (`n≈690–940`), 1 SE ≈ 1.6–1.9 pp.
  Differences below those are noise.

## 7. Diagnostic / falsified scripts (research use only — not part of the benchmark)

These are standalone studies, each resumable, run under the `ego3d` env. They are **not** needed to
reproduce results; they explain *why* the methods behave as they do. **Do not treat their older
"compression improves accuracy" framing as current** — see the verdict in `CLAUDE.md` and
`Notes/Visual-Compression.md`.

| Script | Question it answers |
|---|---|
| `scripts/leverage_diagnostic.py` | go/no-go "体检": effective rank, ridge-leverage Gini, Gram coherence vs a random null (finding: effective rank ≈84 ≪ keep10 budget → selection can only matter at keep ≤5%) |
| `scripts/four_way_extreme.py` | stratified-random vs anchor vs ridge-leverage vs log-det "engine" at keep5/3 (also the shared backbone for §3) |
| `scripts/vision_ablation.py` | does the model use vision at all (real / keep10 / black-image / noise-feature controls) |
| `scripts/redundancy_analysis.py` | do VisPruner-kept tokens have higher cross-view redundancy |
| `scripts/p4_fps_causal.py` | does an explicit low-redundancy / high-coverage kept set causally help |
| `scripts/anchor_spread.py` | how concentrated block-cvsp Layer-1 anchors are across views (finding: anchors already spread → explicit pairing not worth it) |

Each takes `--model_path`, `--n`, `--image_root`, `--vsi_items`/`--tasks`/`--configs` and writes to
`logs/<study>/` or a single JSON; run with `python scripts/<name>.py --help` for specifics.

## 8. Quick gotchas checklist

- ✅ Run from repo root, `ego3d` env, with the two env vars set.
- ✅ Prep VSI-Bench before any `vsi:*` config; keep `--frames` consistent everywhere.
- ✅ a20s40 / block-cvsp / two-stage are in **Harness B** (`cvsp_curve.py` / `qstage_curve.py`),
  **not** in `--compress_method`.
- ✅ For `cvsp`/`block_cvsp` always pass the same `--tag` when re-running, or you won't find/score
  the file.
- ✅ Score Ego3D by **ACC**; RMSE improvements under compression are an artifact.
- ✅ Runs are resumable and deterministic — re-launching continues; an `n=200` file extends to full.
- ✅ Leave `--merge none` (the merge variant failed).
